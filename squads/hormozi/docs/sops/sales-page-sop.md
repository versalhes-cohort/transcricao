## Part 1: Purpose

Build high-converting sales pages that systematically attack every variable of the Value Equation. The 17 Elements framework removes reasons to NOT buy, rather than "selling" the prospect.

**Core Philosophy:**
```
"A pagina nao vende. A pagina remove as razoes para NAO comprar. Grande diferenca."
```

**Value Equation Attack:**
```
Value = (Dream Outcome × Perceived Likelihood) / (Time Delay × Effort & Sacrifice)

17 Elements systematically:
- MAXIMIZE: Dream Outcome + Perceived Likelihood (numerator)
- MINIMIZE: Time Delay + Effort & Sacrifice (denominator)
```

**Business Impact:**
- Above the fold: >40% scroll rate
- Complete page: >3% conversion on cold traffic
- Checkout: <25% abandonment

## Part 2: Scope

### When This SOP Applies
- Creating new sales pages for products/services
- Optimizing existing pages with low conversion
- Building checkout flows
- VSL landing pages
- Long-form sales letters

### When This SOP Does NOT Apply
- Simple opt-in pages (use lead magnet SOP)
- Content-only pages (blogs, articles)
- Branding-focused pages without conversion goal
- Checkout-only pages (handled in Section C only)

### Page Types Covered
- Product sales pages
- Service sales pages
- Course/program sales pages
- High-ticket offer pages
- Low-ticket offer pages

## Part 5: Process Steps

### SECTION A: ABOVE THE FOLD (0-600px) - Os 3 Segundos Criticos

#### Step 1: Craft Headline Principal (O Hook Magnetico)
| Field | Value |
|-------|-------|
| **Description** | Create main headline that maximizes Dream Outcome instantly |
| **Cognitive Type** | Synthesis + Creativity |
| **Automation Potential** | MEDIUM - AI can generate, human selects |
| **Executor** | Hybrid |
| **Precondition** | Avatar pain points and desired outcomes documented |
| **Output** | Headline text (tested variants) |
| **Guardrails** | Must include specific result + timeframe + anti-pain |
| **Decision Rule** | "If headline doesn't stop the scroll, nothing else matters" |

**Formula:**
```
"Como [RESULTADO ESPECIFICO] em [TIMEFRAME]
Sem [DOR #1] ou [DOR #2]"
```

**Quality Spectrum:**
- RUIM: "Curso de Marketing Digital"
- BOM: "Aprenda Marketing Digital"
- MATADOR: "Como Conseguir 30 Clientes High-Ticket em 90 Dias Sem Gastar Fortuna em Ads ou Virar Escravo do Instagram"

**A/B Test Priority:**
- Versao Dor: "Pare de Perder 87% dos Seus Leads"
- Versao Desejo: "Converta 10x Mais com o Mesmo Trafego"

---

#### Step 2: Write Subheadline (A Promessa Expandida)
| Field | Value |
|-------|-------|
| **Description** | Expand promise with mechanism and specificity to increase Perceived Likelihood |
| **Cognitive Type** | Synthesis |
| **Automation Potential** | MEDIUM |
| **Executor** | Hybrid |
| **Precondition** | Headline approved, unique mechanism defined |
| **Output** | Subheadline text |
| **Guardrails** | Must include mechanism + social proof + qualification |
| **Decision Rule** | "Subheadline adds credibility that headline promises" |

**Template:**
```
"O [MECANISMO UNICO] usado por [NUMERO] [AVATAR ESPECIFICO]
para [RESULTADO COMPROVADO] - Agora disponivel para [QUALIFICACAO]"
```

**Example:**
"O Sistema de 3 Fases usado por 4.500+ Donos de Academia para encher suas unidades em 30 dias - Agora disponivel para quem esta cansado de sala vazia"

**Positioning:** Directly below headline, font 60% of headline size.

#### Step 3: Design CTA Principal #1 (O Botao Impossivel de Ignorar)
| Field | Value |
|-------|-------|
| **Description** | Create primary button that reduces Effort & Sacrifice perception |
| **Cognitive Type** | Execution + UX |
| **Automation Potential** | HIGH - Template-based |
| **Executor** | Worker |
| **Precondition** | Headline/subheadline approved |
| **Output** | CTA button with text and specs |
| **Guardrails** | Benefit-oriented text, not action-oriented |
| **Decision Rule** | "Button text = outcome they want, not action they take" |

**Technical Specifications:**
- Color: Minimum 3:1 contrast with background
- Size: Minimum 200px width on mobile
- Text: Benefit-oriented, not action-oriented

**Quality Spectrum:**
- RUIM: "Comprar Agora"
- BOM: "Quero Meu Acesso"
- MATADOR: "Sim! Quero 30 Clientes em 90 Dias ->"

**Psychological Hack:** Use arrow (->) in button. Increases CTR 12-26%.

| Field | Value |
|-------|-------|

| **Automation Potential** | HIGH - Timer/counter automation |

## Part 6: Decision Rules

### DR-001: Section Priority
```yaml
rule: SECTION_PRIORITY
trigger: Building new page
condition:
  - IF above_fold_complete == false THEN "STOP - Complete Section A first"
  - IF section_a_converting < 40% THEN "OPTIMIZE - Section A before B"
  - IF main_body_complete AND checkout_abandon > 25% THEN "FOCUS on Section C"
action: Always Section A > B > C priority
confidence: HIGH (3-second rule)
```

### DR-002: Headline Testing
```yaml
rule: HEADLINE_AB_TEST
trigger: Headline created
condition:
  - ALWAYS test pain vs desire variant
  - IF no clear winner after 500 visitors THEN test specificity
  - IF still no winner THEN test format (question vs statement)
action: Run A/B tests in priority order
confidence: HIGH
```

### DR-003: VSL vs Image Decision
```yaml
rule: MEDIA_TYPE_SELECTION
trigger: Element 6 planning
condition:
  - IF traffic == cold THEN use_vsl(8-15min)
  - IF traffic == warm THEN use_vsl(15-45min) OR image_hero()
  - IF product == visual_heavy THEN image_hero() preferred
  - IF offer == complex THEN use_vsl() preferred
action: Match media to traffic temperature and offer complexity
confidence: HIGH
```

### DR-004: Scarcity Type
```yaml
rule: SCARCITY_SELECTION
trigger: Element 5 planning
condition:
  - IF capacity_limited == true THEN use_spot_counter()
  - IF time_limited == true THEN use_countdown()
  - IF bonus_limited == true THEN use_bonus_expiration()
  - IF no_real_scarcity THEN SKIP_ELEMENT() # Better than fake
action: Only use real scarcity mechanisms
confidence: HIGH
```

### DR-005: Bullet Quantity
```yaml
rule: BENEFIT_BULLET_COUNT
trigger: Element 7 creation
condition:
  - IF bullets < 5 THEN "Add more - not enough value stacking"
  - IF bullets > 7 THEN "Remove weakest - diluting impact"
  - IF bullets == 5-7 THEN "OPTIMAL"
action: Maintain 5-7 bullet range
confidence: HIGH
```

### DR-006: Guarantee Strength
```yaml
rule: GUARANTEE_LEVEL
trigger: Element 13 design
condition:
  - IF product_confidence == high THEN use_triple_guarantee()
  - IF product_confidence == medium THEN use_standard_30day()
  - IF product_confidence == low THEN FIX_PRODUCT_FIRST()
action: Guarantee strength reflects product confidence
confidence: HIGH
```

## Part 7: Exceptions & Edge Cases

### Exception 1: No Testimonials Yet
**Trigger:** New product without customer proof
**Action:** Use alternative proof sources
- Beta tester feedback
- Expert endorsements
- Founder credentials
- Methodology proof (research, case studies)
**Guardrail:** Label appropriately ("Resultados de beta testers")

### Exception 2: Complex Offer Requiring Long Explanation
**Trigger:** Cannot explain value in standard format
**Action:**
- Lead with simplified hook
- Use VSL as primary explainer
- Break value stack into digestible chunks
- Add "How It Works" mini-section

### Exception 3: High-Ticket ($5K+)
**Trigger:** Premium pricing
**Action:**
- Extend Section B with more proof
- Add calendar booking CTA instead of direct purchase
- Include "Not for everyone" positioning strongly
- Consider application requirement before purchase

### Exception 4: Low Traffic Volume
**Trigger:** <500 visitors/month for testing
**Action:**
- Skip extensive A/B testing
- Use best practices defaults
- Rely on qualitative feedback (heatmaps, recordings)
- Test major changes only (headline, offer)

### Exception 5: Mobile-First Audience
**Trigger:** >70% mobile traffic
**Action:**
- Prioritize thumb-zone CTA placement
- Shorter above-fold content
- Larger button sizes (250px+ width)
- Collapsible FAQ sections

## Appendix A: Golden Rules

### Core Philosophy
```
"Show, don't tell" - Para cada claim, mostre prova.
Para cada feature, mostre beneficio.
Para cada objecao, mostre solucao.
```

### Why It Works
Each element systematically attacks a variable of the Value Equation:
- First 5 elements decide if visitor stays or leaves
- Next 9 build the logical and emotional case
- Last 3 remove final friction

### The Meta-Rule
```
"A pagina nao vende. A pagina remove as razoes para NAO comprar."
```

## Appendix B: Technical Specifications

### Above the Fold Requirements
- Height: 0-600px maximum critical content
- Load priority: Headline > CTA > Proof > Urgency
- CTA button: Minimum 200px width (mobile)
- Contrast: 3:1 minimum for CTA

### VSL Specifications
- Aspect ratio: 16:9
- Thumbnail: Play button + benefit text
- Duration by traffic:
  - Cold: 8-15 minutes
  - Warm: 15-45 minutes

### Mobile Optimization
- Button width: 250px+ on mobile
- Font scaling: 16px minimum body
- Thumb zone: CTAs within easy reach
- Collapsible sections for long content
