## Part 1: Purpose

Maximize revenue per customer through strategic upsells - what you offer NEXT after someone buys. This SOP covers the STRATEGY of upselling (types, timing, positioning), while the upsell-page-sop covers the PAGE implementation.

**Core Equation:**
```
Profit Multiplier = Base Offer Profit + (Upsell #1 Profit) + (Upsell #2 Profit) + (Upsell #N Profit)

Example (Hamburger Model):
- Hamburger profit: $0.25
- + Fries profit: $0.75
- + Drink profit: $1.75
- + Supersize profit: $1.00
- TOTAL: $3.00 (11.6x increase from $0.25)
```

**Business Impact:** "Sua primeira oferta nem sempre precisa gerar lucro. Em outras palavras, a coisa que voce mais vende nem sempre e a coisa em que voce mais lucra. Voce lucra na segunda, terceira, e no caso do negocio de hamburguer, quarta ofertas e alem."

**Guiding Philosophy:** "Se o McDonald's nao fizesse upsell de batatas fritas e refrigerante, nao haveria McDonald's. Se voce quer vencer, tem que descobrir sua versao de 'Voce quer batatas fritas com isso?'"

## Part 4: The Four Upsell Types Framework

### Framework Overview

```
+============================================================================+
|                    THE FOUR UPSELL TYPES (Hormozi)                         |
+============================================================================+
|                                                                            |
|  1. CLASSIC UPSELL              2. MENU UPSELL                            |
|  "You can't have X without Y"   "You don't need this, you need that"      |
|  +--------------------------+   +--------------------------+              |
|  | Immediate problem solver |   | Multiple options offered |              |
|  | Right after purchase     |   | Unsell + Prescribe + AB  |              |
|  | Single next step         |   | Card on file close       |              |
|  +--------------------------+   +--------------------------+              |
|                                                                            |
|  3. ANCHOR UPSELL              4. ROLLOVER UPSELL                         |
|  "Premium option first"        "Credit toward next purchase"              |
|  +--------------------------+   +--------------------------+              |
|  | Show expensive first     |   | Apply previous $ to new  |              |
|  | Get the gasp             |   | Re-engage old customers  |              |
|  | Main offer = bargain     |   | Rescue upset customers   |              |
|  +--------------------------+   +--------------------------+              |
|                                                                            |
+============================================================================+
```

### Why Upsells Fail

```
Three Failure Modes:
1. WRONG OFFER: "Voce oferece algo que eles nao querem, muito diferente,
   ou nao resolve o problema deles"

2. WRONG TIMING: "Voce oferece no momento errado, antes deles terem
   experimentado o problema"

3. WRONG PRESENTATION: "Voce oferece da maneira errada, eles nao acreditam
   em voce, ou uma combinacao dos tres"
```

### Upsell Categories

```
Offers tend to be:
1. MORE of what they just received (quantity)
   Example: "Why have one hamburger when you could have two?"

2. BETTER versions (quality)
   Example: "Why have mystery meat when you could have filet?"

3. NEW/COMPLEMENTARY things (different)
   Example: "Would you like fries and a drink with that?"
```

## Part 5: Process Steps

### TYPE I: THE CLASSIC UPSELL

#### Step 1.1: Map Problem-Solution Chain
| Field | Value |
|-------|-------|
| **Description** | Identify what problem your offer reveals and what solves it |
| **Cognitive Type** | Analysis (customer journey mapping) |
| **Automation Potential** | LOW - Requires business understanding |
| **Executor** | Human (strategist) |
| **Precondition** | Deep understanding of customer problems |
| **Output** | Problem-solution chain document |
| **Guardrails** | Each offer must reveal a genuine problem the upsell solves |
| **Decision Rule** | "Sua oferta principal resolve um problema e cria outro. Seu upsell imediatamente resolve esse proximo problema." |

**Structure:** "Voce nao pode ter X sem Y"

**Principle:** "Se um problema aparece e voce pode resolve-lo imediatamente em troca de dinheiro, faca isso."

---

#### Step 1.2: Prioritize by Profit Margin
| Field | Value |
|-------|-------|
| **Description** | Order upsell offers from highest to lowest profit margin |
| **Cognitive Type** | Analysis (financial) |
| **Automation Potential** | HIGH - Spreadsheet calculation |
| **Executor** | Worker |
| **Precondition** | Unit economics known for all products |
| **Output** | Ranked upsell priority list |
| **Guardrails** | Never lead with low-margin upsell if high-margin available |
| **Decision Rule** | "Ofereca upsells mais lucrativos primeiro. Se ofereco dois produtos e um tem lucro maior que o outro, ofereco a opcao de maior lucro primeiro." |

---

#### Step 1.3: Use the "Say No to Say Yes" Technique
| Field | Value |
|-------|-------|
| **Description** | Frame upsell so customer says "no" but actually means "yes" |
| **Cognitive Type** | Synthesis (psychological framing) |
| **Automation Potential** | MEDIUM - Scripts can be templated |
| **Executor** | Hybrid |
| **Precondition** | Upsell identified and priced |
| **Output** | Scripted upsell presentation |
| **Guardrails** | Must be ethical - don't manipulate, facilitate decision |
| **Decision Rule** | "Faca-os dizer nao para dizer sim" |

**Script Template:**
```
"Voce nao quer mais nada, quer?"

Translation: "Voce nao quer mais nada alem do que acabei de oferecer, quer?"
Result: Saying "no" = agreeing to the upsell
```

**Source:** "Sempre fiquei impressionado com a frequencia que o negociante de casacos de pele fazia as pessoas comprarem coisas dizendo nao."

---

#### Step 1.4: Deploy Surprise & Delight
| Field | Value |
|-------|-------|
| **Description** | Give all bonuses even if customer said yes before you offered all |
| **Cognitive Type** | Judgment (customer experience) |
| **Automation Potential** | HIGH - Rule-based fulfillment |
| **Executor** | Worker |
| **Precondition** | Multiple bonuses prepared |
| **Output** | Bonus delivery regardless of negotiation point |
| **Guardrails** | Everyone gets same bonuses to avoid FOMO complaints |
| **Decision Rule** | "Se disserem sim antes de adicionar todos, ainda de todos os quatro" |

**Principle:** "Isso vai surpreende-los e encanta-los. E garante que voce ainda venda a mesma coisa para todos para que ninguem se sinta excluido depois."

---

#### Step 1.5: Capitalize on Hyper-Buying Cycles
| Field | Value |
|-------|-------|
| **Description** | Identify when customers are in peak spending mode |
| **Cognitive Type** | Perception (behavioral patterns) |
| **Automation Potential** | MEDIUM - Data can flag, human confirms |
| **Executor** | Hybrid |
| **Precondition** | Customer purchase history available |
| **Output** | Hyper-buyer identification and upsell timing |
| **Guardrails** | Don't overwhelm, but don't hold back either |
| **Decision Rule** | "Venda mais quando estao comprando mais" |

**Hyper-Buying Triggers:**
- Weddings
- New hobbies
- Having babies
- Moving to new places
- New business ventures
- Health transformations

**Principle:** "E uma janela curta de tempo quando estao mais animados sobre uma coisa nova que vao fazer. E quando gastam uma grande quantidade de dinheiro em um periodo curto."

---

#### Step 1.6: Create Problems with Free Bonuses
| Field | Value |
|-------|-------|
| **Description** | Design bonuses that reveal upsell opportunities |
| **Cognitive Type** | Synthesis (strategic design) |
| **Automation Potential** | LOW - Creative strategy |
| **Executor** | Human |
| **Precondition** | Upsell product/service exists |
| **Output** | Bonus that creates need for upsell |
| **Guardrails** | Bonus must have genuine value, not just setup |
| **Decision Rule** | "Use bonus gratis para criar problemas que ofertas de upsell resolvem" |

**Example - Fur Coat Storage:**
```
Offer: Free ear muffs with coat storage
Upsell: "We'll store these for $30 too. You don't want to store
anything else, do you?" (Customer says no = yes to storage)
Result: Customers pay $30 to store FREE ear muffs
```

#### Step 1.7: Accelerate Access
| Field | Value |
|-------|-------|
| **Description** | Deliver upsell value as fast as possible - instant if possible |
| **Cognitive Type** | Execution |
| **Automation Potential** | MEDIUM - Depends on product type |
| **Executor** | Hybrid |
| **Precondition** | Fulfillment system ready |
| **Output** | Immediate or near-immediate upsell delivery |
| **Guardrails** | If can't deliver instantly, set clear expectations |
| **Decision Rule** | "Quanto mais rapido as pessoas tem acesso as coisas, mais vao valoriza-las" |

**Principle:** "Uma coisa de $10.000 que voce recebe depois vale menos que uma coisa de $10.000 que voce recebe agora."

**Pro Tip:** "Pontos bonus se voce puder coloca-lo em suas maos antes deles terem dito sim. E muito mais dificil devolver algo depois de te-lo recebido."

|-------|-------|

- "Pacote de Resultados Mais Rapidos"

#### Step 1.9: Integrate Upsells Into Delivery
| Field | Value |
|-------|-------|
| **Description** | Make upsell products part of how you deliver other offers |
| **Cognitive Type** | Synthesis (product design) |
| **Automation Potential** | MEDIUM - Design once, automate delivery |
| **Executor** | Hybrid |
| **Precondition** | Core offer delivery method established |
| **Output** | Integrated product experience that reveals upsells |
| **Guardrails** | Integration must feel natural, not forced |
| **Decision Rule** | "Integre upsells em suas outras ofertas" |

**Examples:**
```
- Meal plans include "optional supplement suggestions"
  → Customers ask about supplements → Sell supplements

- Sales training suggests "optional software tools"
  → Gym owners see the tools → Buy the software
```

#### Step 1.10: BAMFAM - Book A Meeting From A Meeting
| Field | Value |
|-------|-------|
| **Description** | Never end an interaction without scheduling the next one |
| **Cognitive Type** | Execution (habit) |
| **Automation Potential** | LOW - Requires conversation |
| **Executor** | Human |
| **Precondition** | Calendar/scheduling system available |
| **Output** | Next meeting scheduled before current one ends |
| **Guardrails** | Treat "I'll get back to you" as objection |
| **Decision Rule** | "Quanto mais vezes voce pode fazer upsell, mais pessoas voce fara upsell" |

**Sharran's Rule:** "Um cliente deve saber a proxima vez que vera voce e por que antes de sair."

**Script:**
```
"Antes de sair, vamos agendar nosso proximo encontro.
Quando funciona melhor para voce - [Opcao A] ou [Opcao B]?"
```

#### Step 1.11: Upsell Guarantees
| Field | Value |
|-------|-------|
| **Description** | Charge for guarantees, extended warranties, and insurance |
| **Cognitive Type** | Synthesis (offer design) |
| **Automation Potential** | HIGH - Standard offer |
| **Executor** | Worker |
| **Precondition** | Core product delivered |
| **Output** | Paid guarantee/warranty upsell |
| **Guardrails** | 5-50% premium on top of base price |
| **Decision Rule** | "Voce pode cobrar por garantias, garantias estendidas e seguros em vez de da-los gratuitamente" |

**Example - Art Studio:**
```
Before: Replaced damaged portraits for free
After: Ask "Would you pay 10% extra for this?"
Result: 30% of customers buy - pure profit
```

### TYPE II: MENU UPSELLS

#### Step 2.1: Unsell What They Don't Need
| Field | Value |
|-------|-------|
| **Description** | Tell customers what they DON'T need to emphasize what they DO |
| **Cognitive Type** | Judgment (customer fit) |
| **Automation Potential** | MEDIUM - Can be scripted for common scenarios |
| **Executor** | Hybrid |
| **Precondition** | Multiple products available |
| **Output** | Crossed-out items that don't apply |
| **Guardrails** | Must genuinely assess fit, not fake unselling |
| **Decision Rule** | "Voce desvende dizendo aos clientes o que nao precisam para que possa enfatizar o que precisam" |

**Examples:**
```
Massagist: "Temos massagem linfatica disponivel, mas voce nao esta
gravida ou acabou de sair de cirurgia, certo? Entao podemos riscar isso."

Dog Food: "Voce nao vai precisar deste saco pequeno ou dessas coisas
de filhote. Voce tem um cachorro grande."
```

**Principle:** "Mais tarde, mantive produtos apenas para risca-los."

---

#### Step 2.2: Prescribe What They Need
| Field | Value |
|-------|-------|
| **Description** | Tell customers exactly what to buy and how to use it |
| **Cognitive Type** | Synthesis (expert recommendation) |
| **Automation Potential** | MEDIUM - Templates for common scenarios |
| **Executor** | Hybrid |
| **Precondition** | Customer situation understood |
| **Output** | Personalized prescription with usage instructions |
| **Guardrails** | Must explain integration with previous purchases |
| **Decision Rule** | "Upsells por prescricao funcionam bem quando oferecer uma escolha e inconveniente e voce so tem uma coisa que resolve o problema" |

**Two Components:**
1. Explain how it integrates with what they already bought
2. Personalize and detail how to maximize value

**Example:**
```
"Voce vai querer dar ao seu cachorro um desses mastigaveis para
articulacoes em cada refeicao. E a cada 90 dias, de a eles um
desses wafers para vermes do coracao."
```

---

#### Step 2.3: Offer A/B Choices
| Field | Value |
|-------|-------|
| **Description** | Ask preference between two options, both of which are purchases |
| **Cognitive Type** | Synthesis (choice architecture) |
| **Automation Potential** | HIGH - Scripted |
| **Executor** | Worker |
| **Precondition** | Multiple similar products available |
| **Output** | Customer chooses between A or B (not yes/no) |
| **Guardrails** | Either choice must result in sale |
| **Decision Rule** | "Em vez de perguntar se clientes querem comprar produto sim ou nao, perguntamos qual produto preferem, A ou B" |

**A/B Categories:**
```
- Quantity: "Uma garrafa ou duas?"
- Start dates: "Comecar amanha ou segunda?"
- Payment: "Dinheiro ou cartao?"
- Flavors: "Chocolate ou baunilha?"
- Times: "Manha ou tarde?"
- Medium: "Ler ou ouvir?"
- Delivery: "Padrao ou expresso?"
- Sizes: "Pequeno ou medio?"
- Colors: "Preto ou branco?"
- Materials: "Papel ou plastico?"
- Personnel: "Joao ou Sara?"
- Communication: "Ligar ou mensagem?"
```

**Pro Tip - Add a Nudge:**
```
"Este e meu favorito"
"X geralmente e uma aposta segura"
"Muitas pessoas adoram isso"
"As sessoes de terca sao um pouco menores se voce gostar disso"
"Amy trabalha muito bem com estudantes do ensino medio"
```

#### Step 2.4: Use Card on File
| Field | Value |
|-------|-------|
| **Description** | Ask if they want to use the card already on file |
| **Cognitive Type** | Execution |
| **Automation Potential** | HIGH |
| **Executor** | Worker |
| **Precondition** | Payment method stored from previous purchase |
| **Output** | Frictionless payment completion |
| **Guardrails** | Must have legitimate card on file |
| **Decision Rule** | "Se voce facilitar para as pessoas comprarem, mais pessoas comprarao" |

**Script:**
```
"Voce quer usar o cartao que tem em arquivo?"
```

**Why It Works:** "Diminui os custos ocultos de comprar, escolher qual cartao usar, tira-lo, ser lembrado de decisoes de compra feias do passado, ate mesmo o incomodo de comprar coisas com pressa."

**For First Purchase:** "Qual cartao voce quer usar?"

|-------|-------|

| **Cognitive Type** | Culture (organizational behavior) |
| **Automation Potential** | LOW - Human behavior |

| **Precondition** | Staff trained on full product range |
| **Output** | Staff recommendations that build trust |
| **Guardrails** | Must genuinely help customer, not fake |

**Principle:** "Encoraje clientes a ajudar clientes a burlar o sistema de proposito. Seus funcionarios tem conhecimento interno. Entao permita que mostrem aos clientes como obter o maximo valor do que voce tem a oferecer."

#### Step 2.6: Handle Sold Out Situations
| Field | Value |
|-------|-------|
| **Description** | Collect payment even if product not immediately available |
| **Cognitive Type** | Execution |
| **Automation Potential** | MEDIUM |
| **Executor** | Hybrid |
| **Precondition** | Product will be available soon |
| **Output** | Sale captured, delivery expectation set |
| **Guardrails** | Must deliver within reasonable timeframe |

### TYPE III: ANCHOR UPSELLS

#### Step 3.1: Present Premium Option First
| Field | Value |
|-------|-------|
| **Description** | Always show the most expensive option before the main offer |
| **Cognitive Type** | Synthesis (psychological pricing) |
| **Automation Potential** | HIGH - Sequence can be automated |
| **Executor** | Worker |
| **Precondition** | Premium version of offer exists |
| **Output** | Premium offer presented first |
| **Guardrails** | Premium must be 5-10x main offer price |
| **Decision Rule** | "Se voce apresentar uma oferta mais cara antes de uma oferta menos cara, mais pessoas comprarao a oferta menos cara do que teriam comprado sozinha" |

**The Suit Story:**
```
Suit #1 shown: $16,000 → Customer gasps
Suit #2 shown: $2,200 → Customer relieved, buys immediately
Result: Customer spent 5x their $500 budget and felt good about it
```

---

#### Step 3.2: Get the Gasp
| Field | Value |
|-------|-------|
| **Description** | Wait for the customer's price shock reaction |
| **Cognitive Type** | Perception (emotional reading) |
| **Automation Potential** | LOW - Requires human judgment |
| **Executor** | Human |
| **Precondition** | Premium price shown |
| **Output** | Customer reaction identified |
| **Guardrails** | Don't rush past the gasp - it's necessary |
| **Decision Rule** | "Quando voce faz um anchor upsell corretamente, os clientes terao um mini ataque de panico. Chamo isso de engasgo." |

**Key Insight:** "Engasgos costumavam me estressar muito quando estava vendendo. Mas entao percebi algo enorme. Quanto maior o engasgo, mais eles compravam."

---

#### Step 3.3: Come to the Rescue
| Field | Value |
|-------|-------|
| **Description** | After gasp, ask if they care about premium features |
| **Cognitive Type** | Judgment (face-saving) |
| **Automation Potential** | MEDIUM - Script can be prepared |
| **Executor** | Hybrid |
| **Precondition** | Gasp received |
| **Output** | Permission to show main offer |
| **Guardrails** | Must genuinely save face, not mock |
| **Decision Rule** | "Uma vez que voce obtem o engasgo, venha ao resgate" |

**Script:**
```
"Voce se importa muito com [premium feature that makes it expensive]?"
Customer: "Nem um pouco."
"Otimo, deixe-me mostrar esta outra opcao..."
```

**Pro Tip:** "Ele ja tinha o terno de um oitavo do preco puxado antes da minha reacao. Ele sabia que eu provavelmente engasgaria."

---

#### Step 3.4: Present Main Offer as the Bargain
| Field | Value |
|-------|-------|
| **Description** | Show main offer which now looks like incredible value |
| **Cognitive Type** | Execution |
| **Automation Potential** | HIGH |
| **Executor** | Worker |
| **Precondition** | Anchor set, gasp received, rescue delivered |
| **Output** | Main offer presented in context of anchor |
| **Guardrails** | Main offer must have same PRIMARY features as premium |
| **Decision Rule** | "Apenas ajuste alguns recursos da sua oferta premium para fazer sua oferta principal" |

**Feature Strategy:**
```
PRIMARY Features (keep same): Core function/outcome
SECONDARY Features (change): Designer, material, timing, access level
```

**Example:**
```
Primary: "It's a suit" (same in both offers)
Secondary: Designer name, fabric quality, custom fit
Result: 1/8 the price for the same primary function
```

### TYPE IV: ROLLOVER UPSELLS

#### Step 4.1: Identify Rollover Candidates
| Field | Value |
|-------|-------|
| **Description** | Determine which customers are eligible for rollover credits |
| **Cognitive Type** | Analysis (segmentation) |
| **Automation Potential** | HIGH - CRM rules |
| **Executor** | Worker |
| **Precondition** | Customer purchase history available |
| **Output** | Segmented rollover candidate lists |
| **Guardrails** | Each segment needs different approach |
| **Decision Rule** | "Rollover upsells creditam parte ou toda a compra anterior de um cliente para sua proxima oferta" |

**Four Candidate Types:**
1. **Old customers** (6+ months since last purchase)
2. **Upset customers** (yours - as alternative to refund)
3. **Competitor's upset customers** (steal them)
4. **Current customers** (immediate upsell)

---

#### Step 4.2: Design Re-engagement Campaigns
| Field | Value |
|-------|-------|
| **Description** | Contact dormant customers with rollover offers |
| **Cognitive Type** | Synthesis (campaign design) |
| **Automation Potential** | MEDIUM - Outreach automated, message personalized |
| **Executor** | Hybrid |
| **Precondition** | Past purchase data, contact info |
| **Output** | Re-engagement campaign with rollover credit |
| **Guardrails** | Credit must be meaningful, not insulting |
| **Decision Rule** | "Entre em contato com clientes antigos, seis meses ou mais desde a ultima compra" |

**Example Script - Chiropractor:**
```
"Oi, Sra. Banks, queria devolver seu dinheiro. Tem um minuto?
Otimo. Sim, queria ver como esta sua dor nas costas.
Oh, sinto muito em ouvir isso.
Bem, tenho boas noticias. Como forma de agradecer, quero dar
$500 do seu dinheiro de volta como credito para ficar livre
de dor para sempre. Isso te interessa?
Otimo, vamos marcar voce."
```

**ROI Story:** "Fiz videos personalizados para 200 clientes passados oferecendo $4.000 de credito para retornar. Conseguimos cerca de 20% das pessoas para aceitar a oferta. Um dia gravando videos nos deu $1,9 milhao extra em receita recorrente anual."

---

#### Step 4.3: Use Rollover Instead of Refund
| Field | Value |
|-------|-------|
| **Description** | Offer upset customers credit toward different/better product |
| **Cognitive Type** | Judgment (customer recovery) |
| **Automation Potential** | LOW - Situation specific |
| **Executor** | Human |
| **Precondition** | Customer requesting refund |
| **Output** | Rollover credit instead of refund |

## Part 7: Exceptions & Edge Cases

### Exception 1: Customer Says "I Need to Think About It"
**Trigger:** Customer hesitates on upsell
**Action:** Use BAMFAM - schedule follow-up before letting them go
**Script:**
```
"Entendo completamente. Vamos agendar uma ligacao rapida para
amanha para que eu possa responder qualquer duvida.
Manha ou tarde funciona melhor?"
```

### Exception 2: Product Out of Stock
**Trigger:** Upsell product unavailable
**Response:** Collect payment, set delivery expectations
**Script:**
```
"Este item esta temporariamente esgotado, mas posso reservar
agora e enviamos em [X dias]. Assim voce garante o preco atual."
```

### Exception 3: Customer Already Has Everything
**Trigger:** Customer doesn't need typical upsells
**Action:** Offer quantity upgrades or referral incentives
**Options:**
- "Quer levar duas?"
- "Conhece alguem que se beneficiaria?"
- Offer premium support tier

### Exception 4: Anchor Price Doesn't Get a Gasp
**Trigger:** Customer seems comfortable with premium price
**Analysis:** They may actually want to buy it
**Action:** Don't rush to cheaper option - close the premium
**Script:**
```
"Parece que essa opcao premium faz sentido para voce.
Quer usar o cartao em arquivo?"
```

### Exception 5: Refund Request Received
**Trigger:** Customer wants money back
**Action:** Always offer rollover before refund
**Script:**
```
"Antes de processarmos o reembolso, posso oferecer uma opcao?
Esse valor pode ser creditado para [alternativa] que pode
resolver melhor seu problema. Interessado em saber mais?"
```
