## Part 1: Purpose

Create product, offer, and brand names that command attention, communicate value instantly, and convert at higher rates. Naming is the act of making your offer appear NEW and UNIQUE even if the underlying methodology is similar to competitors.

**Core Definition:**
```
Naming = Making something appear NEW and UNIQUE
Good Naming = Prospect believes "this might be the thing that works for me"
Bad Naming = Prospect associates with past failures or generic commodity
```

**Why Naming Matters:**
```
"People want coisas novas e unicas que resolvam seus problemas."
(Cap. 16, $100M Ofertas)

If your name sounds like everyone else:
- You become a COMMODITY
- You compete on PRICE only
- Past failures transfer to your offer
```

**The Rebrand Effect:**
```
Same Product + New Name = New Opportunity in Prospect's Mind

"Se voce vendeu um programa de perda de peso a alguem que nao funcionou
dois anos atras, mas voce mudou o programa, melhorou e agora sabe que
funciona, renomea-lo algo diferente fara com que a mesma pessoa considere
comprar de novo." (Cap. 16)
```

**Business Impact:**
```
Strong Name enables:
1) Perceived uniqueness vs competitors
2) Premium pricing justification (name signals value)
3) Escape from commodity pricing
4) Recovery from past failed offers
5) Memory hooks for word-of-mouth

"O nome nao faz o produto. O nome faz o marketing do produto
10X mais facil ou 10X mais dificil." (Swipefile Master, Conclusao)
```

## Part 2: Scope

### When This SOP Applies
- Naming a new product or offer
- Rebranding a failed or stale offer
- Creating bonus/component names within an offer
- Naming programs, courses, memberships
- Naming high-ticket coaching/consulting packages

### When This SOP Does NOT Apply
- Company-level branding (see branding-sop.md)
- Logo/visual identity design
- Domain name availability (separate technical process)
- Trademark legal review (requires legal counsel)

### Business Types
- Info products and courses
- Coaching and consulting programs
- SaaS and software products
- Physical products (DTC, supplements, consumer goods)
- Services and service packages
- B2B and B2C

## Part 4: The MAGICO Naming Framework

### Framework Overview (From $100M Offers, Cap. 16)

```
MAGICO = Six Components for Powerful Names

M = Mecanismo unico (unique mechanism - the "how")
A = Alvo atraente (attractive target - the outcome)
G = Grafia diferente (different spelling - creative format)
I = Ideia que ajuda (idea that helps - solution hint)
C = Convergencia temporal (time convergence - deadline)
O = Objetivo final (final goal - dream result)
```

### MAGICO Breakdown

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    MAGICO NAMING COMPONENTS                              │
├────────────┬────────────────────────────────────────────────────────────┤
│ COMPONENT  │ FUNCTION & EXAMPLES                                        │
├────────────┼────────────────────────────────────────────────────────────┤
│ M - Mech.  │ The unique "how" - differentiates method                   │
│            │ Ex: "Sistema", "Metodo", "Formula", "Blueprint"            │
├────────────┼────────────────────────────────────────────────────────────┤
│ A - Alvo   │ The target/who it's for OR target outcome                  │
│            │ Ex: "Six Pack", "Millionaire", "High Performance"          │
├────────────┼────────────────────────────────────────────────────────────┤
│ G - Grafia │ Creative spelling/formatting that stands out               │
│            │ Ex: "10X", "P90X", "6em7", unusual caps                    │
├────────────┼────────────────────────────────────────────────────────────┤
│ I - Ideia  │ The solution concept/theme                                 │
│            │ Ex: "Accelerator", "Intensive", "Breakthrough"             │
├────────────┼────────────────────────────────────────────────────────────┤
│ C - Conv.  │ Time element - creates urgency and specificity             │
│            │ Ex: "6 Semanas", "90 Dias", "48 Horas", "5 Minutos"        │
├────────────┼────────────────────────────────────────────────────────────┤
│ O - Obj.   │ The dream result/final goal                                │
│            │ Ex: "Fit", "Rich", "Free", "Authority"                     │
└────────────┴────────────────────────────────────────────────────────────┘
```

### MAGICO Example (From $100M Offers)

```
"O Sistema de 6 Semanas Six Pack FRAQUEZA-A-RIQUEZA"

M = Sistema (mechanism - structured approach)
A = Six Pack (attractive target - specific outcome)
G = FRAQUEZA-A-RIQUEZA (creative spelling/format)
I = (embedded in transformation theme)
C = 6 Semanas (time convergence)
O = Six Pack + Riqueza (dream result)
```

## Part 6: Decision Rules

### DR-001: Program vs Product Terminology
```yaml
rule: TERMINOLOGY_SELECTION
trigger: Naming a service or deliverable
condition:
  - IF service-based THEN use "Program" (acceptance feeling)
  - IF product-based THEN use "System" or "Method"
action: Choose term that reduces sales resistance
confidence: HIGH (Cap. 16: "Prefiro usar 'programa' em vez de 'produto'")
rationale: "Da a impressao de que e abrangente e que eles estao sendo aceitos"
```

### DR-002: Specificity Selection
```yaml
rule: SPECIFICITY_OVER_GENERIC
trigger: Choosing between general and specific claims
condition:
  - IF can be specific THEN always choose specific
  - IF generic necessary THEN add one specific element
action: Maximum specificity that is truthful
confidence: HIGH (Cap. 16)
rationale: "Quanto mais especifico, mais real se torna para o prospect"
```

### DR-003: Archetype Selection
```yaml
rule: ARCHETYPE_MATCH
trigger: Selecting naming archetype
condition:
  - IF non-technical_audience AND complex_category THEN "LITERAL_EXTREMO"
  - IF creating_new_behavior THEN "VERBO_CULTURAL"
  - IF emotional_transformation THEN "ESTADO_DESEJADO"
  - IF time_quantity_differentiator THEN "NUMERICO_ESPECIFICO"
  - IF commodity_category THEN "PARADOXO_INTRIGANTE"
  - IF community_focus THEN "TRIBAL_IDENTIFIER"
  - IF visual_physical_benefit THEN "METAFORA_VISUAL"
  - IF premium_pricing THEN "STATUS_SIGNAL"
  - IF new_category THEN "INVENTOR_FRIENDLY"
  - IF trust_barrier THEN "HERITAGE_FABRICADO"
action: Select matching archetype
confidence: HIGH (Swipefile Master analysis)
```

### DR-004: Price Tier Naming
```yaml
rule: PRICE_NAME_ALIGNMENT
trigger: Naming offer at specific price point
condition:
  - IF price < $100 THEN use_urgency_words (Quick, Instant, Hack)
  - IF price $100-$1000 THEN use_value_words (System, Formula, Accelerator)
  - IF price $1000-$10000 THEN use_transformation_words (Intensive, Certification)
  - IF price > $10000 THEN use_status_words (Inner Circle, Mastermind, Partnership)
action: Match naming language to price psychology
confidence: HIGH (Swipefile price tier analysis)
```

### DR-005: Rebrand Trigger
```yaml
rule: WHEN_TO_RENAME
trigger: Evaluating existing offer performance
condition:
  - IF associated_with_past_failure THEN rename_required
  - IF commodity_perception THEN rename_required
  - IF market_confused THEN rename_required
  - IF version_2.0_insufficient THEN full_rename_required
action: Execute full MAGICO rename process
confidence: HIGH (Cap. 16)
rationale: "Uma nova oportunidade e...bem...nova e talvez esta funcione"
```

### DR-006: The Seven Laws Application
```yaml
rule: SEVEN_IMMUTABLE_LAWS
trigger: Final name validation
checks:
  - LAW_1: Simplicidade > Criatividade
  - LAW_2: Beneficio > Feature
  - LAW_3: Emocao > Logica
  - LAW_4: Especifico > Generico
```

### Exception 1: The "Version 2.0" Trap
**Trigger:** Considering adding version number to existing name
**Analysis:** "Se voce disser o mesmo nome ou mesmo uma versao 2.0, eles associarao com 'nao funcionou'" (Cap. 16)
**Response:** Full rename required, not version increment

### Exception 2: Extreme Specificity Backfire
**Trigger:** Specific claim might not be achievable by all customers
**Analysis:** "Voce tem que ser capaz de FAZER TUDO QUE VOCE PROMETE. Sempre diga a verdade." (Cap. 16)
**Response:** Be creative about truth, but never lie. Use "up to" or case study framing.

### Exception 3: Cultural/Language Barriers
**Trigger:** Name must work across multiple languages
**Analysis:** Test for unintended meanings in target markets
**Response:** Score 8+ on "funciona em 3+ idiomas" before proceeding

### Exception 4: Category Creator Names
**Trigger:** Creating entirely new category
**Analysis:** "Inicio de categoria: Nomes educacionais necessarios" (Swipefile)
**Response:** Use descriptive elements even if less creative

### Exception 5: High-Competition Commodity
**Trigger:** Entering saturated market with similar products
**Analysis:** "Categoria madura: Paradoxos e controversos funcionam" (Swipefile)
**Response:** Consider paradox archetype (Liquid Death approach)

```
LEI 1: SIMPLICIDADE > CRIATIVIDADE

"Pessoas compram sentimentos."

LEI 5: CURTO > LONGO
"Media dos unicornios: 7.3 caracteres."

LEI 6: HISTORIA > DESCRICAO
Warby Parker (personagens literarios) > Affordable Eyewear Direct

LEI 7: TIMING > PERFEICAO
"Nome 80% bom lancado hoje > nome 100% perfeito nunca lancado"
```
