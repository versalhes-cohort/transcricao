## Part 1: Purpose

Create landing pages that remove every reason NOT to buy. A landing page doesn't sell - it removes all objections and friction from the decision process.

**Core Equation (Value Equation applied to LP):**
```
Value = (Dream Outcome × Perceived Likelihood) / (Time Delay × Effort & Sacrifice)

Landing Page Elements Attack Each Variable:
├── Dream Outcome ↑: Headlines, benefícios, cases
├── Probabilidade Percebida ↑: Prova social, garantias, autoridade
├── Atraso Temporal ↓: Promessas de velocidade, vitórias rápidas
└── Esforço & Sacrifício ↓: UX simples, suporte visível, checkout fácil
```

**Business Impact:** "Uma landing page que converte 5% com tráfego qualificado vale mais que uma que converte 10% com tráfego lixo." The page can literally be the difference between bankruptcy and financial freedom.

## Part 2: Scope

### When This SOP Applies
- Creating new landing pages for lead capture
- Optimizing existing landing pages
- Building sales pages for offers
- Any page designed for single-action conversion

### When This SOP Does NOT Apply
- General website pages (about, contact, etc.)
- Content/blog pages
- Multi-product catalog pages
- Non-conversion focused pages

### Business Types
- Any business with online offers
- Lead magnet capture pages
- High-ticket and low-ticket sales
- B2B and B2C

### The 10 Phases Framework

```
Phase 1: Above the Fold (0-3 segundos)
Phase 2: Experiência Móvel (60%+ do tráfego)
Phase 3: Estrutura de Persuasão
Phase 4: Qualificação e Objeções
Phase 5: Estrutura de Valor
Phase 6: Autoridade e Confiança
Phase 7: Conversão e Checkout
Phase 8: Otimização e Monitoramento
Phase 9: Suporte e Recuperação
Phase 10: Compliance e Proteção
```

## Part 5: Process Steps

### PHASE 1: ABOVE THE FOLD (0-3 SEGUNDOS)

#### Step 1.1: Headline Magnética
| Field | Value |
|-------|-------|
| **Description** | Create headline using structure "Como [RESULTADO ESPECÍFICO] em [TIMEFRAME] sem [DOR PRIMÁRIA]" |
| **Cognitive Type** | Synthesis (crafting compelling promise) |
| **Automation Potential** | LOW - Requires human creativity |
| **Executor** | Human (copywriter) |
| **Precondition** | Avatar pains and desires documented |
| **Output** | Headline that makes visitor think "como assim?" or "isso é possível?" |
| **Guardrails** | Teste Obrigatório: Leia em voz alta - se não rolar natural, está errada |
| **Decision Rule** | Must be first thing visible, largest font on page |

**Benchmark:** Deve fazer o visitante pensar "como assim?" ou "isso é possível?"

#### Step 1.2: Subheadline de Credibilidade
| Field | Value |
|-------|-------|
| **Description** | Create subheadline using formula "O [MECANISMO] usado por [NÚMERO] [AVATAR] para [RESULTADO COMPROVADO]" |
| **Cognitive Type** | Synthesis |
| **Automation Potential** | LOW |
| **Executor** | Human |
| **Precondition** | Proof metrics available (number of users, results) |
| **Output** | Subheadline increasing Perceived Likelihood |
| **Guardrails** | Red Flag: Se parece com jargão de vendedor, refaça |
| **Decision Rule** | Fonte 60% menor que headline, diretamente abaixo |

#### Step 1.3: CTA Principal
| Field | Value |
|-------|-------|
| **Description** | Create benefit-oriented CTA button |
| **Cognitive Type** | Perception (customer psychology) |
| **Automation Potential** | MEDIUM - Template-based |
| **Executor** | Hybrid |
| **Precondition** | Desired outcome clearly defined |
| **Output** | CTA orientado a benefício ("Quero 30 Clientes em 90 Dias") vs. ação ("Comprar") |
| **Guardrails** | Design: Mínimo 200px largura mobile, contraste 3:1, cor que "grita" |
| **Decision Rule** | Within above-the-fold, visível sem scroll. Hack: Use seta (→) - aumenta CTR 12-26% |

#### Step 1.4: Prova Social Imediata
| Field | Value |
|-------|-------|
| **Description** | Display immediate trust indicators |
| **Cognitive Type** | Analysis (selecting best proof) |
| **Automation Potential** | MEDIUM |
| **Executor** | Hybrid |
| **Precondition** | Real metrics available |
| **Output** | Format: "⭐⭐⭐⭐⭐ X+ Alunos \| Y% Satisfação \| Resultados em Z dias" |
| **Guardrails** | Números reais vs. estimativas - sempre números reais |
| **Decision Rule** | Logo abaixo do CTA ou integrada ao header |

#### Step 1.5: Indicador de Urgência
| Field | Value |
|-------|-------|
| **Description** | Add real urgency element |
| **Cognitive Type** | Judgment |
| **Automation Potential** | HIGH - Timer automation |
| **Executor** | Worker (technical) + Human (strategy) |
| **Precondition** | Real urgency exists (not fake) |
| **Output** | Timer regressivo, vagas limitadas, or bônus expirando |
| **Guardrails** | CRÍTICO: Deve ser 100% real - falsa escassez destrói trust de longo prazo |
| **Decision Rule** | Próximo ao CTA ou em barra fixa no topo. Cor vermelha ou laranja |

## Part 6: Decision Rules

### DR-001: Above the Fold Completeness
```yaml
rule: ABOVE_FOLD_CHECK
trigger: Page review
condition:
  - IF headline_visible == false THEN "FAIL"
  - IF subheadline_visible == false THEN "FAIL"
  - IF cta_visible == false THEN "FAIL"
  - IF proof_visible == false THEN "WARNING"
  - IF urgency_visible == false THEN "WARNING"
action: All 5 elements must be visible without scroll
confidence: HIGH (Source: Phase 1)
```

### DR-002: Headline Quality Test
```yaml
rule: HEADLINE_NATURAL_TEST
trigger: Headline draft complete
condition:
  - IF read_aloud_feels_natural == false THEN "REVISE"
  - IF visitor_thinks_how_possible == false THEN "REVISE"
  - IF looks_like_sales_jargon == true THEN "REVISE"
action: Read aloud test mandatory
confidence: HIGH (Source: 1.1, 1.2)
```

### DR-003: Mobile First
```yaml
rule: MOBILE_PERFORMANCE
trigger: Page deployment
condition:
  - IF load_time > 3_seconds THEN "FAIL"
  - IF pagespeed_score < 90 THEN "WARNING"
  - IF text_size < 16px THEN "FAIL"
  - IF button_height < 44px THEN "FAIL"
action: Fix mobile issues before launching
confidence: HIGH (Source: Phase 2)
```

### DR-004: Benefit vs Feature
```yaml
rule: BENEFIT_NOT_FEATURE
trigger: Bullet copy review
condition:
  - IF bullet_is_feature THEN transform_to_benefit()
  - IF missing_emotional_benefit THEN add_emotional_benefit()
  - IF bullets > 7 THEN reduce_to_7()
action: Apply formula "✓ [RESULTADO] para que você possa [BENEFÍCIO EMOCIONAL]"
confidence: HIGH (Source: 3.3, Erro Fatal #1)
```

### DR-005: Single Offer Rule
```yaml
rule: ONE_PAGE_ONE_OFFER
trigger: Page structure review
condition:
  - IF offers_count > 1 THEN "FAIL - Erro Fatal #2"
action: One landing page = one offer = one decision
rationale: "Confused mind says no. Excited mind says yes."
confidence: HIGH (Source: Erro Fatal #2)
```

### DR-006: Guarantee Strength
```yaml
rule: GUARANTEE_QUALITY
trigger: Guarantee section review
condition:
  - IF guarantee_text == "Satisfação garantida" THEN "WEAK - Erro Fatal #3"
  - IF guarantee == null THEN "CRITICAL - Erro Fatal #3"
  - IF guarantee_is_specific == true THEN "PASS"
action: Create specific, conditional, even "unfair" guarantee
confidence: HIGH (Source: 5.3, Erro Fatal #3)
```

### DR-007: Price Context
```yaml
rule: PRICE_WITH_CONTEXT
trigger: Price reveal
condition:
  - IF value_shown_before_price == false THEN "FAIL - Erro Fatal #4"
  - IF comparisons_absent == true THEN "WARNING"
  - IF guarantee_absent == true THEN "WARNING"
action: Always show value → comparisons → price → guarantee → urgency
```

## Part 7: Exceptions & Edge Cases

### Exception 1: Warm Traffic Pages
**Trigger:** Audience already knows you
**Adjustment:**
- VSL can be longer (15-45 min vs 8-15 min for cold)
- Higher conversion benchmarks (>8% vs >3% cold)
- Can reduce proof elements
- Can be more direct with offer

### Exception 2: Lead Magnet vs Sales Page
**Trigger:** Free offer capture
**Adjustment:**
- Form fields: Minimum (name + email only)
- No price comparison needed
- Guarantee = quality of free content
- Exit-intent: Different free asset

### Exception 3: High-Ticket Offers
**Trigger:** Price >R$5,000
**Adjustment:**
- Add application step (qualification)
- More authority elements needed
- More cases required
- May need sales call CTA instead of direct purchase

### Exception 4: Technical/B2B Audience
**Trigger:** Sophisticated buyer
**Adjustment:**
- More specific claims
- More detailed proof
- Less emotional, more logical
- ROI-focused copy

### Exception 5: Regulated Industries
**Trigger:** Health, finance, legal
**Adjustment:**
- Extensive disclaimers mandatory
- Claims must be verified
- Legal review required
- May need compliance approval

## Appendix A: The 5 Fatal Errors

### Erro Fatal #1: Vender Recursos em Vez de Resultados
**O Que Vejo:** "10 módulos + 50 templates + grupo no Facebook"
**Por Que Mata:** Ninguém compra brocas, compram buracos.
**A Correção:** Para cada feature, complete: "...para que você possa [RESULTADO EMOCIONAL]"

**Exemplo:**
- ❌ "Acesso ao grupo VIP no Facebook"
- ✅ "Rede de networking com outros donos de agência faturando 6-7 dígitos para que você nunca mais se sinta isolado construindo seu império"

### Erro Fatal #2: Múltiplas Ofertas na Mesma Página
**O Que Vejo:** "Curso básico por R$297 OU curso avançado por R$997 OU consultoria por R$5000"
**Por Que Mata:** Paradoxo da escolha. Quanto mais opções, menor a conversão.
**A Correção:** Uma landing page para cada oferta.
**Regra Hormozi:** "Confused mind says no. Excited mind says yes."

### Erro Fatal #3: Garantia Fraca ou Inexistente
**O Que Vejo:** "Satisfação garantida" ou pior: nenhuma garantia.
**Por Que Mata:** Sem reversão de risco, a Probabilidade Percebida permanece baixa.
**A Correção:** Garantia específica, condicional e até "unfair":
```
GARANTIA TRIPLA "VITÓRIA CERTA":
✓ Implemente por 21 dias. Sem resultados? Dinheiro de volta.
✓ Não ficar satisfeito por qualquer motivo? Dinheiro de volta.
✓ Aplicar tudo e não ter ROI positivo? Devolvemos 2X seu investimento.
```

### Erro Fatal #4: Preço Sem Contexto
**O Que Vejo:** "Investimento: R$997" - solto, sem ancoragem.
**Por Que Mata:** Preço sem contexto é sempre "caro".
**A Correção:** Sequência matadora:
1. Mostre o stack de valor (R$15.000 total)
2. Compare com alternativas (consultor cobra R$10k)
3. Apresente seu preço (apenas R$1.997)
4. Reforce com garantia (sem risco)
5. Adicione urgência (apenas hoje)

### Erro Fatal #5: Copy Genérico de Nicho
**O Que Vejo:** "Transforme sua vida", "Alcance seus sonhos", "Método comprovado"
**Por Que Mata:** Linguagem genérica não ressoa com ninguém.
**A Correção:** Use a linguagem EXATA que seu avatar usa.

**Exemplo Real (Academia):**
- ❌ Genérico: "Aumente sua receita"
- ✅ Específico: "Pare de brigar por sócio com mensalidade de R$49 quando você pode ter lista de espera cobrando R$149"
