## Pre-Implementation Checklist

### Offer Readiness

- [ ] Core offer delivers real value (passes "Value Equation" test)
- [ ] Offer is already working at baseline (don't add scarcity to broken offer)
- [ ] Target audience is defined and reachable
- [ ] Sales process is documented and functional

### Capacity Analysis

- [ ] True operational capacity is known (not guessed)
- [ ] Current utilization percentage is calculated
- [ ] Weekly/monthly onboarding capacity is mapped
- [ ] Delivery quality at capacity is confirmed

### Infrastructure Ready

- [ ] Calendar system can enforce time windows
- [ ] CRM can track capacity and waitlist
- [ ] Communication channels are configured (email, social, landing pages)
- [ ] Countdown timer tools are available (if using urgency)

### Ethical Commitment

- [ ] Leadership commits to honoring ALL stated limits
- [ ] Team understands "no exceptions" policy for deadlines
- [ ] Sell-out announcement process is defined
- [ ] Waitlist management is planned

## Scarcity Type Selection Checklist

### Physical Product Scarcity

> "Ter lançamentos limitados é um método testado e comprovado"

- [ ] Identified release type (flavor, color, design, size)
- [ ] Quantity set BELOW expected demand
- [ ] "Always sell out" principle accepted
- [ ] Sell-out announcement planned
- [ ] Frequency set (monthly optimal)

**Go/No-Go:** Can you honestly limit production and commit to selling out?

### Total Business Limit Scarcity

> "Minha agência atenderá apenas vinte e cinco clientes no total"

- [ ] Maximum client capacity determined
- [ ] Service quality at capacity validated
- [ ] Waitlist system configured
- [ ] Periodic expansion plan (10-20%) outlined
- [ ] "Always leave some demand unmet" accepted

**Go/No-Go:** Can you turn away clients when at capacity?

### Growth Rate Limit Scarcity

> "Aceitamos apenas 5 novos clientes por semana"

- [ ] Weekly/monthly new client capacity determined
- [ ] Openings remaining tracking enabled
- [ ] Sales team trained on scarcity scripts
- [ ] Buffer for uncertainty maintained

**Go/No-Go:** Can you honestly report openings remaining?

### Cohort Limit Scarcity

> "Aceitamos 100 clientes 4 vezes por ano"

- [ ] Cohort size set below expected demand
- [ ] Cohort start dates defined
- [ ] Open/close enrollment process documented
- [ ] Compound tracking initialized

**Go/No-Go:** Can you close enrollment even with demand?

### Premium Access Scarcity (Extreme)

> "Venda uma oferta muito limitada de acesso 1-a-1"

- [ ] Delivery method you don't hate identified
- [ ] Tiny limit set (3-5 typical)
- [ ] Premium price determined
- [ ] Communication plan for availability

**Go/No-Go:** Are you prepared for top 1% of 1% clients?

### Exit Barrier Scarcity

> "Se eles saírem, nunca poderão retornar"

- [ ] Confirmed this is a SMALL group application
- [ ] Policy clearly communicated before enrollment
- [ ] Enforcement mechanism in place
- [ ] Understood this "loses teeth" with larger groups

**Go/No-Go:** Can you enforce the "no return" policy?

## Urgency Type Selection Checklist

### Cohort-Based Urgency

> "Quanto menos frequentemente você inicia novos clientes, mais poderoso"

- [ ] Cohort start dates are fixed and real
- [ ] Basic script prepared: "Sign up today = Monday group"
- [ ] Advanced script prepared: "Cancellation slot available"
- [ ] Late-join handling defined (catch-up or runway extension)

**Go/No-Go:** Are your start dates real and enforced?

### Seasonal/Promotional Urgency

> "A promoção real pode ser a mesma, mas nomeá-la algo diferente"

- [ ] Promotion calendar mapped for the year
- [ ] Each promotion has unique name (New Year, Valentine's, Spring, etc.)
- [ ] Landing pages with visible dates ready
- [ ] Ads with deadline copy prepared
- [ ] Local business frequency (monthly) considered if applicable

**Go/No-Go:** Are you committed to ending promotions on stated dates?

### Price/Bonus Urgency

> "Cria medo de perder a promoção, em vez do serviço real"

- [ ] Identified what expires (discount, bonus, installation, workshop)
- [ ] Expiration is honest (you WILL remove after date)
- [ ] Interchangeable elements ready for rotation
- [ ] Script prepared: "Not sure how long we'll run this one"

**Go/No-Go:** Will you actually remove the bonus/discount after deadline?

### Price Increase Urgency

> "Nunca aumente seus preços sem avisar as pessoas"

- [ ] Price increase is ACTUALLY happening
- [ ] Announcement timeline defined
- [ ] Pipeline clearing campaign planned
- [ ] New price is set and locked in

**Go/No-Go:** Is the price increase real and permanent?

### Exploding Opportunity Urgency

> "Cada segundo que alguém atrasa, eles perdem ganhos desproporcionais"

- [ ] Opportunity is genuinely time-sensitive
- [ ] Delay consequences are real and significant
- [ ] Emphasis plan for urgency messaging
- [ ] Evidence of time decay available

**Go/No-Go:** Does waiting actually cost the prospect significantly?

## Combined Deployment Checklist

### Stacking Scarcity + Urgency

> "Frequentemente, escassez e urgência são usadas juntas"

- [ ] Both limits can be honestly enforced
- [ ] Combined message is clear, not overwhelming
- [ ] Example ready: "20 spots, enrollment closes Friday"
- [ ] Both tracking systems active

### Sell-Out Announcement Protocol

> "Você também deve deixar todos saberem que esgotou"

- [ ] Announcement template ready
- [ ] Channels for announcement identified (email, social, landing page)
- [ ] Waitlist capture integrated into announcement
- [ ] Next opportunity date estimated and communicated

## Communication Checklist

### Scarcity Messaging

- [ ] Capacity percentage visible: "Currently at X% capacity"
- [ ] Spots remaining visible: "Only Y spots left"
- [ ] Waitlist mentioned: "Join waitlist for next opening"
- [ ] Social proof implied: "X people chose to work with us"

### Urgency Messaging

- [ ] Deadline prominently displayed
- [ ] Countdown timer active (if digital)
- [ ] Reminder sequence configured
- [ ] Last-day messaging prepared

### Scripts Ready

#### Scarcity Scripts:
- [ ] Total limit: "We only serve X clients total"
- [ ] Growth rate: "We accept only X new clients per week, Y spots filled"
- [ ] Cohort: "We accept X clients Y times per year, doors open then close"
- [ ] Premium: "I have X spots for direct access at [price]"

#### Urgency Scripts:
- [ ] Cohort deadline: "Sign up today = next group Monday"
- [ ] Seasonal: "Our [Name] Promo ends [Date]"
- [ ] Price/Bonus: "Take advantage of the discount, we change every 4 weeks"
- [ ] Price increase: "Prices going up, get in now!"

## Ethical Enforcement Checklist

### Before Launch

- [ ] All limits are real and verifiable
- [ ] Deadline enforcement is committed (no exceptions)
- [ ] Team trained on "no exception" policy
- [ ] Honest scarcity as fallback if needed

### During Campaign

- [ ] Capacity tracking is real-time accurate
- [ ] Deadlines are visibly counting down
- [ ] No fake urgency ("only 2 left!" when unlimited)
- [ ] Customer challenges handled with proof

### At Capacity/Deadline

- [ ] Campaign closed as stated (no extensions)
- [ ] Sell-out announced publicly
- [ ] Late requests redirected to next opportunity
- [ ] Waitlist capturing overflow demand

### After Campaign

- [ ] All claims were honored
- [ ] No credibility damage identified
- [ ] Compound tracking updated
- [ ] Learnings documented for next campaign

## Quality Gate Checkpoints

### QG-001: Honesty Gate (BLOCKING)

Before ANY scarcity-urgency deployment:

- [ ] Limit is real and enforceable
- [ ] Deadline is real and will be honored
- [ ] No "fake scarcity" elements

**If failed:** Use honest scarcity or do not deploy

### QG-002: Sell-Out Gate (WARNING)

At end of limited campaign:

- [ ] Campaign sold out (or close to it)

**If failed:** Reduce limit next time

### QG-003: Deadline Enforcement Gate (BLOCKING)

When late request comes in:

- [ ] Deadline was honored (no exceptions)
- [ ] Customer redirected to next opportunity

**If failed:** Escalate - credibility at risk

### QG-004: Announcement Gate (WARNING)

After sell-out:

- [ ] Sell-out was announced publicly

**If failed:** Announce immediately - this is part of the strategy

### QG-005: Credibility Gate (BLOCKING)

Campaign review:

- [ ] No fake scarcity claims made
- [ ] All limits were honored
- [ ] No deadline exceptions granted

**If failed:** Audit and correct before next campaign

### QG-006: Compound Effect Gate (WARNING)

Quarterly review:

- [ ] Effectiveness improving or stable
- [ ] Time-to-sell-out not increasing
- [ ] Waitlist not shrinking

**If failed:** Strategy refresh required

## Metrics Tracking Checklist

### Primary Metrics

- [ ] Sell-out rate tracked (target: >80%)
- [ ] Decision speed tracked (target: faster than baseline)
- [ ] Last-day sales percentage tracked (benchmark: 30-60%)

### Secondary Metrics

- [ ] Waitlist conversion rate
- [ ] Price objection frequency
- [ ] Repeat buyer rate post-exposure
- [ ] Time-to-sell-out trend

### Compound Effect Tracking

- [ ] Campaign-over-campaign comparison active
- [ ] Quarterly trend analysis scheduled
- [ ] Improvement/decline patterns monitored

## Post-Campaign Review Checklist

### Performance Review

- [ ] Did we sell out? (If no, limit was too high)
- [ ] Did last-day spike occur? (If no, urgency needs work)
- [ ] Were all limits honored? (If no, credibility risk)
- [ ] Was sell-out announced? (If no, missed opportunity)

### Credibility Review

- [ ] Any customer complaints about fake scarcity?
- [ ] Any deadline exceptions granted?
- [ ] Any capacity violations?
- [ ] Public perception intact?

### Compound Effect Review

- [ ] Is sell-out time improving?
- [ ] Is waitlist growing?
- [ ] Are price objections decreasing?
- [ ] Are repeat purchases increasing?

### Optimization Notes

- [ ] What worked well?
- [ ] What needs improvement?
- [ ] Limit adjustments for next time?
- [ ] Urgency messaging refinements?

## Quick Reference: Do's and Don'ts

### DO:

- Set limits BELOW expected demand
- ALWAYS sell out (better than oversupply)
- ALWAYS announce sell-outs
- Enforce deadlines without exception
- Use honest scarcity as default
- Track compound effectiveness over time
- Vary marketing more often (especially local businesses)

### DON'T:

- Use fake scarcity or urgency
- Reopen after deadline closes
- Make exceptions for late requests
- Over-order and fail to sell out
- Skip the sell-out announcement
- Use the same fake countdown everyone uses
- Damage credibility for short-term gains

## Final Pre-Launch Validation

Before launching any scarcity-urgency campaign, confirm:

1. **Honesty Check:** Every limit and deadline is real and enforceable
2. **Enforcement Commitment:** Team commits to zero exceptions
3. **Announcement Ready:** Sell-out announcement template prepared
4. **Tracking Active:** All metrics and compound tracking initialized
5. **Waitlist Configured:** Overflow demand can be captured
6. **Scripts Ready:** Sales team has all necessary scripts

**Launch Authorization:**
- [ ] All items above confirmed
- [ ] Strategy owner sign-off
- [ ] Campaign can proceed
