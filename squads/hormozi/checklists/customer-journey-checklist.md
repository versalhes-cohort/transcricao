# Customer Journey 4-Milestone Checklist

---

## Milestone 1: ACTIVATE

```yaml
milestone_activate:
  definition:
    - [ ] Activation point claramente definido
    - [ ] Métrica específica estabelecida
    - [ ] Target timeline definido (ex: primeiros 30 dias)

  tactics:
    - [ ] Onboarding 100% direcionado ao activation
    - [ ] Quick wins no caminho identificados
    - [ ] Fricções mapeadas e removidas
    - [ ] Celebração definida para quando atinge

  unlockable:
    - [ ] Incentivo por atingir activation definido
    - [ ] Tipo: unlock, bonus, badge, access
    - [ ] Entrega automática ou manual definida
```

---

## Milestone 2: TESTIMONIAL

```yaml
milestone_testimonial:
  timing:
    - [ ] Pedido vem IMEDIATAMENTE após activation
    - [ ] Momento de máxima satisfação identificado
    - [ ] Não esperar = não perder o momento

  capture_method:
    - [ ] Método primário definido (video, written, screenshot)
    - [ ] Método secundário para quem não quer video
    - [ ] Duração esperada (1-2 min video)

  questions_prepared:
    - [ ] "Onde você estava antes?"
    - [ ] "O que mudou?"
    - [ ] "Qual foi o momento decisivo?"
    - [ ] "O que você diria para alguém considerando?"

  template_ready:
    - [ ] Script de pedido documentado
    - [ ] Incentivo por participação definido
    - [ ] Processo de follow-up se não responder
```

---

## Milestone 3: REFER

```yaml
milestone_refer:
  timing:
    - [ ] Pedido vem LOGO APÓS testimonial
    - [ ] Ou após grande resultado/satisfação
    - [ ] Estado emocional positivo = melhor momento

  request:
    - [ ] Script documentado
    - [ ] "Você conhece alguém passando pelo mesmo?"
    - [ ] Naturalidade, não pressão

  facilitation:
    - [ ] Template de mensagem para enviar
    - [ ] Link/código fácil de compartilhar
    - [ ] Sistema de tracking de referrals

  incentives:
    - [ ] Incentivo para referrer definido
    - [ ] Incentivo para referred definido
    - [ ] Double-sided incentive configurado
```

---

## Milestone 4: ASCEND

```yaml
milestone_ascend:
  trigger_events:
    - [ ] Após activation point atingido
    - [ ] Após 3 meses de tenure
    - [ ] Após resultados significativos
    - [ ] Quando expressa nova necessidade

  offers:
    - [ ] Pelo menos 1 ascension offer definida
    - [ ] Pricing documentado
    - [ ] Value add claro

  positioning:
    - [ ] Próximo passo natural, não venda forçada
    - [ ] Conectado a nova necessidade do cliente

  when_not_to:
    - [ ] Não quando frustrados
    - [ ] Não antes de atingir valor na compra atual
    - [ ] Não durante problemas não resolvidos
```

## Journey Timeline

```yaml
timeline:
  mapped:
    - [ ] Semana 0-4: ACTIVATE - período definido
    - [ ] Semana 4-8: TESTIMONIAL - período definido
    - [ ] Semana 8-12: REFER - período definido
    - [ ] Mês 3+: ASCEND - triggers definidos

  realistic:
    - [ ] Timeline atingível para maioria dos clientes
    - [ ] Buffer para variação individual
```

---

## Unlockables System

```yaml
unlockables:
  per_milestone:
    - [ ] ACTIVATE: Unlock definido (ex: módulo avançado)
    - [ ] TESTIMONIAL: Unlock definido (ex: badge/certificado)
    - [ ] REFER: Unlock definido (ex: desconto/crédito)
    - [ ] ASCEND: Unlock definido (ex: acesso VIP)

  delivery:
    - [ ] Método de entrega definido (auto vs manual)
    - [ ] Notificação ao cliente configurada
```

---

## Tracking & Metrics

```yaml
metrics:
  per_milestone:
    - [ ] % que atinge ACTIVATE
    - [ ] % que dá TESTIMONIAL
    - [ ] % que faz REFER
    - [ ] % que faz ASCEND

  timing:
    - [ ] Tempo médio para cada milestone
    - [ ] Variação identificada
```

```yaml
output_complete:

    - [ ] Scripts para cada pedido (testimonial, referral)

    - [ ] Timeline visual criado

  alignment:
    - [ ] Alinhado com onboarding sequence
    - [ ] Alinhado com engagement scoring
```

- **PASS**: >= 81 points (80%)

### If FAIL
The customer journey has critical gaps in one or more milestones. Prioritize completing the ACTIVATE milestone first (it gates all downstream milestones), then ensure each subsequent milestone has clear triggers, scripts, and unlockables before rolling out.
