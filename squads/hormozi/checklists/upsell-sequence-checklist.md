
---

## Type 1: Classic Upsell

```yaml
classic_upsell:
  structure:
    - [ ] "Não pode ter X sem Y" - problema identificado
    - [ ] Solução natural para o problema
    - [ ] Preço definido
    - [ ] Timing definido

  tactics_applied:
    - [ ] Upsells mais lucrativos primeiro
    - [ ] "Você não quer mais nada, quer?" (Não = Sim)
    - [ ] Surprise and delight - dá TODOS bônus
    - [ ] BAMFAM considerado (Book A Meeting From A Meeting)

  script:
    - [ ] Script documentado
    - [ ] Conecta problema → solução
```

---

## Type 2: Menu Upsell

```yaml
menu_upsell:
  4_steps:
    - [ ] UNSELL: O que riscar (não precisam)
    - [ ] PRESCRIBE: O que precisam
    - [ ] A/B: Pergunta sem sim/não
    - [ ] CARD ON FILE: "Usar cartão em arquivo?"

  options:
    - [ ] 2-3 opções definidas
    - [ ] Cada opção com preço e includes
    - [ ] The Economist Play considerado (A, B, A+B)

  scripts:
    - [ ] Unselling script documentado
    - [ ] Prescription script documentado
    - [ ] A/B question documentada
```

---

## Type 3: Anchor Upsell

```yaml
anchor_upsell:
  5_steps:
    - [ ] Âncora definida (5-10x principal)
    - [ ] Âncora é oferta REAL
    - [ ] Rescue question preparada
    - [ ] Oferta principal clara
    - [ ] Pergunta de pagamento

  critical_rule:
    - [ ] Preparado para entregar âncora se aceitarem
    - [ ] Não é oferta falsa

  expected:
    - [ ] % que compra âncora estimado
    - [ ] % que compra principal estimado
```

---

## Type 4: Rollover Upsell

```yaml
rollover_upsell:
  situations:
    - [ ] REENGAJAR clientes antigos (6+ meses)
    - [ ] RESGATAR clientes chateados
    - [ ] ROUBAR de concorrentes
    - [ ] UPSELL clientes atuais

  pricing:
    - [ ] Upsell ≥ 4x crédito
    - [ ] Máximo 25% desconto efetivo
    - [ ] Crédito claramente comunicado

  script:
    - [ ] Script documentado para cada situação
```

## Sequence Order

```yaml
sequence:
  defined:
    - [ ] Ordem dos upsells definida
    - [ ] Timing de cada upsell documentado
    - [ ] Transições naturais

  no_overlap:
    - [ ] Upsells não competem entre si
    - [ ] Cada um resolve problema diferente
```

## Projected Impact

```yaml
impact:
  calculated:
    - [ ] Current avg order value
    - [ ] Projected avg order value
    - [ ] Multiplier esperado (típico: 2-4x)

  realistic:
    - [ ] Conversões baseadas em benchmarks
    - [ ] Não assume 100% aceitação
```
