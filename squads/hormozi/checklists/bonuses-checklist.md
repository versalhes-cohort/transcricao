## Bonus Creation Checklist

### For Each Bonus Asset

#### Classification
- [ ] Is this HIGH VALUE? (Wow factor - something they shouldn't miss)
- [ ] Is this LOW COST to deliver? (Marginal cost near zero)
- [ ] Can it stand alone as distinct from core offer?
- [ ] Is it short in duration but high in quality?
- [ ] Would it get lost if bundled in main offer?

#### Type Priority (check which applies)
- [ ] TOOL/CALCULATOR - Highest value (lowest customer effort)
- [ ] CHECKLIST - High value
- [ ] TEMPLATE/SCRIPT - High value
- [ ] SWIPE FILE - High value
- [ ] RECORDING (workshop/webinar) - Medium-high value
- [ ] ADDITIONAL TRAINING - Lower value (convert to tool if possible)

#### Obstacle Mapping
- [ ] Maps to specific customer obstacle or limiting belief
- [ ] Proves their limiting belief is incorrect
- [ ] OR solves the next logical problem they'll face
- [ ] Documented which objection this bonus addresses

## Bonus Presentation Checklist

### For Each Bonus

#### 1. Name (Benefit in Title)
- [ ] Name includes specific benefit/outcome
- [ ] Name communicates transformation, not just the thing
- [ ] Name is memorable and distinct
- [ ] Name passes the "can you understand the value without explanation?" test

**Name Formula:** [Outcome/Benefit] + [Descriptive Name]

Examples:
- "Foolproof Budget Grocery System" (not "Shopping List")
- "Never-Quit Accountability System" (not "Weekly Check-ins")
- "Live-It-Up-While-You-Slim-Down Eating Out System" (not "Restaurant Guide")

#### 2. Problem Connection
- [ ] Explains how bonus relates to their specific problem
- [ ] Uses their language from the obstacle list
- [ ] Connection is clear and immediate

#### 3. What It Is
- [ ] Clearly describes what they receive
- [ ] Specific and tangible (not abstract)
- [ ] Format is clear (PDF, video, access, etc.)

#### 4. Origin Story
- [ ] Includes how you discovered or created it
- [ ] Authentic and relatable backstory
- [ ] Shows effort/investment that went into creation

#### 5. Future Pacing (Value Equation Aligned)
- [ ] Paints vivid picture of life with bonus benefits
- [ ] At least ONE of these addressed:
  - [ ] FASTER - achieve results in less time
  - [ ] EASIER - simpler process
  - [ ] LESS SACRIFICE - keep what they value
  - [ ] HIGHER LIKELIHOOD - more certain to succeed

#### 6. Proof Element
- [ ] Includes at least ONE of:
  - [ ] Relevant statistic
  - [ ] Client testimonial/case study
  - [ ] Personal experience/result
- [ ] Proof is credible and verifiable

#### 7. Price Anchor
- [ ] Standalone price assigned
- [ ] Price justified (comparison, time savings, cost savings, etc.)
- [ ] Price is believable but impressive

## Bonus Enhancement Checklist

### Scarcity Application (When Appropriate)
- [ ] EXCLUSIVE ACCESS: "Only available through this program"
  - Script: "Only people who sign up for [program] will have access to [bonus] that is never for sale or available anywhere else"
- [ ] LIMITED QUANTITY: Physical or practical limits exist
  - Script: "I have [X] remaining [items], if you buy this program, you can get one as a bonus"

### Urgency Application (When Appropriate)
- [ ] TIME-LIMITED: Bonus only available if they act today
  - Script: "If you buy today, I will add [bonus] which normally costs $[X], for free. I do this because I want to reward the action-takers."

**Key Distinction:**
- Scarcity = Not time-limited, but exclusively available through this offer
- Urgency = Time-limited, buy TODAY or lose the bonus

## Partner Bonus Checklist

### Partner Identification
- [ ] NOT a direct competitor
- [ ] Serves same customer demographic
- [ ] Solves "the next natural thing they'll want"
- [ ] Has products/services with clear perceived value

### Partner Negotiation

#### Initial Trade
- [ ] Pitched exposure to your best prospects (paying customers)
- [ ] Secured product/service to include as bonus
- [ ] Value of partner bonus documented
- [ ] Agreement confirmed (written)

#### Commission Deal (Advanced)
- [ ] Referral commission discussed
- [ ] Rate agreed ($ per person or % of sale)
- [ ] Tracking mechanism established
- [ ] Agreement confirmed (written)

#### Group Discount (Advanced)
- [ ] Exclusive discount negotiated for your customers
- [ ] Discount they couldn't get on their own
- [ ] Creates triple-win structure

**Triple Win Validation:**
- [ ] Your customers win (exclusive access/discount)
- [ ] Partner wins (free marketing to qualified buyers)
- [ ] You win (better offer + potential commission revenue)

## 1-on-1 Sales Bonus Checklist

### Before Presenting Bonuses
- [ ] Full value presentation of core offer completed
- [ ] First close attempt made WITHOUT mentioning bonuses

### If They Say YES
- [ ] Revealed bonuses as surprise AFTER sign-up
- [ ] Created "wow" experience
- [ ] Reinforced their buying decision

### If They Say NO
- [ ] Identified their specific objection
- [ ] Selected bonus that directly addresses that objection
- [ ] Presented bonus with agreement: "Fair enough?"
- [ ] If still no, added another matching bonus
- [ ] NEVER offered a discount (add bonuses instead)

## Final Stack Validation Checklist

### Value Ratio
- [ ] Total bonus value EXCEEDS main offer price
- [ ] Target: Bonus value 2x-5x the main offer price
- [ ] Psychology validated: "If these are the bonuses, the main thing must be even MORE valuable"

### Composition
- [ ] Tools and checklists make up >70% of bonuses
- [ ] All major customer obstacles addressed
- [ ] Each bonus has unique purpose (no redundancy)
- [ ] Partner bonuses included where available

### Individual Bonus Validation
For EACH bonus in the stack:
- [ ] Name has benefit in title
- [ ] Maps to specific obstacle
- [ ] Has proof element
- [ ] Price is justified
- [ ] Delivery method is clear

### Presentation Order
- [ ] Sequenced for maximum psychological impact
- [ ] Highest-impact bonuses prominently featured
- [ ] Scarcity/urgency applied to appropriate bonuses
- [ ] Value builds progressively through stack

## Quick Reference: Bonus Creation Formula

```
1. INVENTORY existing assets (recordings, tools, templates)
2. MAP to customer obstacles
3. NAME with benefit in title
4. CONNECT to their specific problem
5. DESCRIBE what it is
6. TELL origin story
7. FUTURE PACE the benefits (faster/easier/less sacrifice/more likely)
8. PROVE it works (stat/testimonial/experience)
9. PRICE and justify
10. ENHANCE with scarcity/urgency (where appropriate)
11. STACK until value eclipses main offer
```
