# 30-Day Onboarding Checklist

> Checklist de validação para `design-onboarding-sequence.md` (HZ_RET_004)
> Source: $100M Retention Playbook - Step 2 Churn Checklist
> Agent: hormozi-retention

---

## Strategic Foundation

```yaml
foundation:
  activation_point:
    - [ ] Activation point claramente definido
    - [ ] Métrica de sucesso estabelecida
    - [ ] Target achievement rate definido

  format_decision:
    - [ ] Customization level escolhido (custom | generic)
    - [ ] Delivery method escolhido (personal | group | self-serve)
    - [ ] Medium escolhido (live | recorded | hybrid)
    - [ ] Eficácia tier identificado (1-7 scale)
```

---

## Day 0: Welcome & Setup

```yaml
day_0:
  welcome:
    - [ ] Welcome message formato definido (call, video, message)
    - [ ] Conteúdo personalizado ou template pronto
    - [ ] Tom celebratório e encorajador

  roadmap:
    - [ ] Visual timeline criado
    - [ ] Milestones claros (Day 1-3, 7, 14, 30)
    - [ ] Expectativas setadas

  communication:
    - [ ] Canais de comunicação definidos
    - [ ] Frequência de contato estabelecida
    - [ ] Responsável atribuído

  first_task:
    - [ ] Task simples definida
    - [ ] Completável em ≤15 minutos
    - [ ] Cria momentum imediato
```

---

## Days 1-3: Activation Push

```yaml
days_1_3:
  activation_focus:
    - [ ] Direcionamento claro ao activation point
    - [ ] Fricções removidas do caminho
    - [ ] Caminho simplificado

  quick_win:
    - [ ] Vitória rápida definida
    - [ ] Atingível em 24-48h
    - [ ] Celebração planejada (público ou privado)

  homework:
    - [ ] Task específica atribuída
    - [ ] Deadline de 7 dias
    - [ ] Conexão clara com activation point
```

---

## Days 3-7: First Check-In

```yaml
days_3_7:
  check_in:
    - [ ] Formato de check-in definido (call | message)
    - [ ] Perguntas preparadas:
      - "Como está o progresso?"
      - "Alguma dificuldade?"
      - "O que posso ajudar?"

  friction_removal:
    - [ ] Processo para identificar obstáculos
    - [ ] Capacidade de resolver imediatamente
    - [ ] Documentação para melhorar processo

  celebration:
    - [ ] Reconhecimento de progresso
    - [ ] Encorajamento para próximos passos
```

---

## Days 7-14: Connection & Proof

```yaml
days_7_14:
  success_connection:
    - [ ] Método de conectar com membros bem-sucedidos
    - [ ] Formato definido (intro, group call, thread)
    - [ ] Goal: Ver que resultado é possível

  proof_sharing:
    - [ ] Cases de pessoas similares preparados
    - [ ] Timeline de quando atingiram activation
    - [ ] Resultados documentados

  second_check_in:
    - [ ] Avaliar % do caminho ao activation point
    - [ ] Ajustar plano se necessário
```

---

## Days 14-21: Community Integration

```yaml
days_14_21:
  community_events:
    - [ ] Próximos eventos apresentados
    - [ ] Participação encorajada
    - [ ] Valor da comunidade comunicado

  meaningful_connections:
    - [ ] Meta: 2-3 conexões significativas
    - [ ] Introduções manuais se necessário
    - [ ] Tipos: similar goal, achieved result, potential partner
```

---

## Days 21-30: Review & Celebrate

```yaml
days_21_30:
  progress_review:
    - [ ] Call estruturada agendada
    - [ ] Cobertura: onde começou, onde está, o que conquistou

  celebration:
    - [ ] Celebração pública se atingiu activation ou progresso
    - [ ] Formato: community post, shoutout, badge

  testimonial:
    - [ ] Pedido de testimonial no momento de satisfação
    - [ ] Formato: video, written, screenshot

  unlockables:
    - [ ] Próximos benefícios apresentados
    - [ ] Razão para continuar clara
```

---

## Incentive Structure

```yaml
incentives:
  aligned_with_churn:
    - [ ] Churn points identificados
    - [ ] Incentivos posicionados APÓS churn points
    - [ ] Tipos de incentivos definidos
```

---

## Paid Onboarding Option (Se Aplicável)

```yaml
paid_onboarding:
  if_offered:
    - [ ] Nome definido (Accelerator, Fast Start, VIP Setup)
    - [ ] Preço definido (1-3x mensalidade típico)
    - [ ] Inclui diferencial claro vs standard
    - [ ] Posicionamento: "Para quem quer resultado mais rápido"
```

---

## Output Quality

```yaml
output_quality:
  complete_sequence:
    - [ ] Todos os 6 períodos documentados
    - [ ] Scripts/templates para cada touchpoint
    - [ ] Owners atribuídos
    - [ ] Métricas de sucesso definidas
    - [ ] Processo de ajuste se não atinge activation
```

## Scoring

| Category | Items | Points Each | Max |
|----------|-------|-------------|-----|
| Critical — Strategic Foundation, Day 0 First Task, Days 1-3 Activation Push | 14 | 3 | 42 |
| Standard — Days 3-7 Check-In, Days 7-14 Proof, Days 14-21 Community, Days 21-30 Review | 18 | 2 | 36 |
| Bonus — Incentive Structure, Paid Onboarding Option, Output Quality | 10 | 1 | 10 |
| **TOTAL** | **42** | | **88** |

### Thresholds
- **PASS**: >= 71 points (80%)
- **CONDITIONAL**: >= 53 points (60%)
- **FAIL**: < 53 points

### If FAIL
The onboarding sequence will not drive customers to the activation point within 30 days. Prioritize defining the activation point metric and building the Day 0-3 push sequence first -- early momentum is the strongest predictor of long-term retention.
