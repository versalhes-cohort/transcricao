## Pillar II: Speed Checklist

### Speed to First Contact
- [ ] Real-time lead notification system active
- [ ] Alert triggers within 60 seconds of opt-in
- [ ] Rep response time tracking enabled
- [ ] 5-minute contact SLA enforced
- [ ] "That was fast!" feedback tracked
- [ ] Contact time logged per lead
- [ ] 391% conversion increase target (60-second contact)
- [ ] 7x qualification increase target (vs 1 hour later)
- [ ] 78% first responder win rate targeted

### Day 1 Contact Cadence
- [ ] Call within 5 minutes of opt-in
- [ ] Double-dial immediately after first call
- [ ] Voicemail script prepared and used
- [ ] Text sent immediately after voicemail
- [ ] Second double-dial + text (hours apart)
- [ ] Third double-dial + text (hours apart)
- [ ] Total 6+ contact attempts on Day 1

### Days 2-3 Cadence
- [ ] Morning call scheduled
- [ ] Evening call scheduled
- [ ] Text after second call each day
- [ ] Total 4+ attempts per day

### Days 4-7 Cadence
- [ ] One call per day scheduled
- [ ] One text per day scheduled
- [ ] Pattern maintained through Day 7
- [ ] Total 7+ attempts days 4-7

### Day 8+ Transition
- [ ] Long-term nurture sequence activated
- [ ] Content soft CTAs configured
- [ ] Re-engagement triggers defined
- [ ] 90-day proof cycle scheduled

### Speed to First Appointment
- [ ] Same-day appointments offered first
- [ ] 72-hour maximum scheduling window enforced
- [ ] >5 day appointments discouraged
- [ ] Pull-forward protocol active

### Five Appointment Outcomes Prioritized
- [ ] Outcome 1: Close them NOW (talking right now = 100% show)
- [ ] Outcome 2: Hot handoff to active closer
- [ ] Outcome 3: Pull appointment forward to today
- [ ] Outcome 4: Confirm within 72h window
- [ ] Outcome 5: Remove unqualified from calendar

### Pull Forward Script Execution
- [ ] Script memorized by reps:
  - [ ] Opening with recorded line disclosure
  - [ ] Confirmation of scheduled time
  - [ ] "Is now a terrible time?" question
  - [ ] Qualification questions (WHO, WHAT, WHAT tried)
  - [ ] "Good news/Better news" transition
  - [ ] Cancellation slot offer
  - [ ] "Unless now is a good time?" always asked
- [ ] Pull forward success rate tracked
- [ ] Calendar gaps used for pull-forward calls

### Hot Handoff Protocol
- [ ] Edification script memorized:
  - [ ] "You're in luck" opening
  - [ ] Expert positioning (years in game, results)
  - [ ] "We're fortunate to have him"
  - [ ] Calendar check pause
  - [ ] "Slide you in" or "Confirmed" transition
  - [ ] Text introduction promise
  - [ ] Phone number confirmation
- [ ] 3-way text group creation enabled
- [ ] Qualification points communicated
- [ ] Handoff success rate tracked

### Response Speed Standards
- [ ] All channels monitored continuously
- [ ] Response time <5 minutes target
- [ ] Rep response times tracked
- [ ] Slow response alerts configured
- [ ] "We prioritize you" messaging reinforced
- [ ] "Slow response = we don't prioritize you" understood

## Pillar III: Personalization Checklist

### Multi-Channel Outreach
- [ ] Simultaneous blast protocol active:
  - [ ] Call initiated
  - [ ] Text sent
  - [ ] DM sent (if available)
  - [ ] Email sent
- [ ] "Continue where they respond" rule followed
- [ ] Escalation to richer medium when appropriate
- [ ] "Reach out like you MEAN it" attitude enforced

### Lead Qualification
- [ ] Qualification questions defined
- [ ] Lead scoring performed before appointment
- [ ] Poor fits removed from calendar
- [ ] Resources/downsell offered to unqualified
- [ ] "Canceling bad = more time for good" understood

### Lead Scoring Actions
- [ ] GREEN leads: Priority routing to best closer
- [ ] YELLOW leads: Standard process ("garbage man" opportunity)
- [ ] RED leads: Remove from calendar, offer resources
- [ ] "Yellows are the new gold" culture established

### Lead-Closer Routing Implementation
- [ ] Step 1: Best customers analyzed (demographics + actions)
- [ ] Step 2: Scoring questions in opt-in forms
- [ ] Step 3: 1-5 or Red/Yellow/Green scores assigned
- [ ] Step 4: Best leads routed to best closers
- [ ] Step 5: Conversion by pairing tracked
- [ ] Continuous optimization active

### Segmented Messaging (Easy Version)
- [ ] Lead profile research (5 min minimum for high-value):
  - [ ] Website reviewed
  - [ ] Social profiles checked
  - [ ] Company information gathered
  - [ ] Pain points identified
- [ ] Personalized opening crafted
- [ ] Specific observation included
- [ ] Example: "Hey Greg, I see that your site traffic is down 18%..."

### Segmented Messaging (Hard Version)
- [ ] Different email sequences per lead category
- [ ] Different text sequences per lead category
- [ ] Templates matched to demographics
- [ ] Results examples matched to industry

### Push Incentives (Gifts - Reciprocity)
- [ ] Gift card value appropriate (not insulting)
- [ ] Gift card delivery system ready
- [ ] Script prepared: "make sure you're caffeinated..."
- [ ] Gift delivered regardless of show
- [ ] Reciprocity effect measured
- [ ] Timing: night before appointment

### Pull Incentives (A/B Choice - Commitment)
- [ ] First-time client script ready
- [ ] A/B options defined:
  - [ ] Option A clearly described
  - [ ] Option B clearly described
- [ ] Response triggers commitment tracking
- [ ] Implicit commitment logged
- [ ] "Would you prefer X or Y?" training complete
- [ ] Choice categories available:
  - [ ] Quantity (one bottle or two?)
  - [ ] Start dates (tomorrow or Monday?)
  - [ ] Type (shirt or shorts?)
  - [ ] Colors (black or white?)
  - [ ] Personnel (John or Sara?)
  - [ ] Times (morning or afternoon?)

### Proof Delivery
- [ ] Proof timing configured:
  - [ ] After schedule, before show
  - [ ] Between first and second call
  - [ ] Every 90 days for ghosted leads
- [ ] Demographic matching enforced
- [ ] "Show them what they missed" messaging ready
- [ ] Results of others who signed up when they ghosted prepared

## Pillar IV: Volume Checklist

### Contact Attempt Tracking
- [ ] Day 1: 6+ attempts logged
- [ ] Days 2-3: 4+ attempts per day logged
- [ ] Days 4-7: 2+ attempts per day logged
- [ ] Total 15+ attempts in first week
- [ ] Long-term transition after 7 days

### Automated Reminders
- [ ] Immediate confirmation configured:
  - [ ] Time confirmed
  - [ ] Date confirmed
  - [ ] Caller number included
  - [ ] Person meeting with named
- [ ] 24-hour reminder configured
- [ ] 12-hour reminder configured
- [ ] 3-hour reminder configured
- [ ] Transparency about automation maintained
- [ ] Template: "[Auto msg] Hey [Name]! Quick reminder..."
- [ ] Local time zone used

### Manual Reminders
- [ ] Night before personal check-in scheduled
- [ ] Morning of confirmation scheduled
- [ ] 60 minutes before "see you soon" scheduled
- [ ] Real cell phone used (not system number)
- [ ] iPhone preferred (blue messages)
- [ ] Personal touch maintained

### A-C-A Conversation Framework
- [ ] Acknowledge training complete (respond to what they said)
- [ ] Compliment training complete (something genuine)
- [ ] Ask training complete (the next question)
- [ ] Framework applied in all communications
- [ ] Qualify or pull forward through A-C-A

### BAMFAM Protocol
- [ ] "Never end call without scheduling next" rule enforced
- [ ] Script memorized:
  - [ ] "Let me pull up my calendar..."
  - [ ] "[Day A] or [Day B]?" question
  - [ ] "Morning or afternoon?" question
  - [ ] Specific time locked in
  - [ ] Confirmation sent immediately
- [ ] "I'll get back to you" treated as objection
- [ ] "I'll follow up later" = lead goes to no-man's land (avoided)
- [ ] Dodgy about next call = objection to handle
- [ ] BAMFAM success rate tracked
- [ ] "BAMFAM as a way of life" culture established

## Decision Rules Validation

### DR-001: Speed Classification
- [ ] Contact time logged for every lead
- [ ] <60 seconds flagged as EXCEPTIONAL (+391% conversion)
- [ ] <5 minutes flagged as STANDARD
- [ ] >5 minutes flagged as SLOW (revenue leak)
- [ ] Speed reports generated daily
- [ ] Immediate contact prioritized

### DR-002: Scheduling Window
- [ ] Same-day slots offered first always (highest show rate)
- [ ] <=72 hours flagged as ACCEPTABLE
- [ ] >72 hours triggers WARNING (increased no-show risk)
- [ ] >5 days triggers DISCOURAGE alert
- [ ] Scheduling window distribution tracked

### DR-003: Lead Score Routing
- [ ] GREEN leads auto-route to top closers
- [ ] YELLOW leads follow standard process
- [ ] RED leads removed + resources offered
- [ ] "Yellows are the new gold" culture maintained
- [ ] Routing effectiveness measured

### DR-004: Ghost Recovery
- [ ] No response AND hours_until_appointment <24h triggers double-book slot
- [ ] No response AND days_since_schedule >3 triggers proof send
- [ ] Complete ghost adds to 90-day nurture
- [ ] "Show them what they missed" campaign active
- [ ] Ghost recovery rate tracked
- [ ] Never give up completely

### DR-005: Incentive Deployment
- [ ] First-time clients receive push incentive (gift card)
- [ ] Qualified leads receive pull incentive (A/B choice)
- [ ] High-value leads receive both
- [ ] Incentive effectiveness measured
- [ ] ROI per incentive type calculated
- [ ] Reciprocity and commitment created

### DR-006: Volume Escalation
- [ ] Day 1 AND attempts <6 continues attempting
- [ ] Day <=3 AND attempts <10 continues attempting
- [ ] Day <=7 AND attempts <15 continues attempting
- [ ] Day >7 transitions to long-term nurture
- [ ] Contact attempt compliance tracked
- [ ] Front-loaded contact attempts enforced

## Exception Handling

### Exception 1: Too Many Bad Appointments
- [ ] Calendar quality monitored
- [ ] Trigger: Calendar filled with unqualified leads
- [ ] Friction increase protocol ready:
  - [ ] Video watch requirement before scheduler
  - [ ] Sales letter above scheduler
  - [ ] Price visibility added
  - [ ] Scheduler appearance delayed
- [ ] Bad appointment threshold defined
- [ ] Intervention timeline established

### Exception 2: Lead Can't Schedule Within 72h
- [ ] Treated as objection, not fact
- [ ] Script ready: "What days were you planning on [activity related to offer]?"
- [ ] Response: "This appointment is the first step to that"
- [ ] Alternative scheduling explored
- [ ] Persistence maintained professionally

### Exception 3: High Show Rate, Low Close Rate
- [ ] Rep lead quality distribution reviewed
- [ ] Cherry-picking detection active
- [ ] May be selecting easy leads
- [ ] Fair distribution enforced
- [ ] Close rate by lead type analyzed

### Exception 4: Lead Requests Different Closer
- [ ] Accommodation process defined
- [ ] Pattern investigation triggered
- [ ] Customer preference logged
- [ ] If possible, accommodate
- [ ] Root cause analysis conducted
