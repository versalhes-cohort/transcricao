# CBC Readiness Checklist
Clone: @wes-kao (Tier 1)

Validate Cohort-Based Course readiness before launch.

## 1. Course Design Foundation

### 1.1 State Change Clarity
- [ ] "Before" state clearly defined
- [ ] "After" state clearly defined
- [ ] Transformation is specific and measurable
- [ ] Gap between states is achievable in cohort duration

### 1.2 Outcome Promise
- [ ] One clear outcome promised
- [ ] Outcome is desirable to target audience
- [ ] Outcome is believable/credible
- [ ] Outcome differentiates from alternatives

## 2. Course Mechanics (12 Levers)

### 2.1 Structure
- [ ] Cohort duration set (4-8 weeks typical)
- [ ] Cohort size determined (20-50 ideal)
- [ ] Start/end dates fixed
- [ ] Curriculum mapped to weeks

### 2.2 Live Components
- [ ] Live sessions scheduled
- [ ] Session format designed (workshop vs lecture)
- [ ] Interaction points planned
- [ ] Recording/replay policy set

### 2.3 Async Components
- [ ] Pre-work defined per session
- [ ] Assignments designed
- [ ] Resources/readings curated
- [ ] Time commitment clearly communicated

### 2.4 Social Learning
- [ ] Peer interaction format chosen
- [ ] Small group/buddy system if applicable
- [ ] Discussion prompts prepared
- [ ] Community platform ready

### 2.5 Accountability
- [ ] Public commitment mechanism exists
- [ ] Progress visibility designed
- [ ] Check-in cadence set
- [ ] Consequences for non-participation defined

### 2.6 Instructor Presence
- [ ] Office hours scheduled
- [ ] Feedback turnaround time set
- [ ] Q&A responsiveness commitment
- [ ] Personal touch points planned

## 3. Completion Rate Optimization

### 3.1 Onboarding
- [ ] Welcome sequence designed
- [ ] Expectations clearly set
- [ ] Technical setup instructions provided
- [ ] Community introduction facilitated

### 3.2 Week 1 Engagement
- [ ] Quick win in first 48 hours
- [ ] First assignment is low-friction
- [ ] Social connections started early
- [ ] Momentum building activities planned

### 3.3 Drop-off Prevention
- [ ] Known drop-off points addressed
- [ ] Re-engagement triggers set
- [ ] 1:1 outreach for at-risk students
- [ ] "Catch up" mechanisms available

### 3.4 Finish Strong
- [ ] Celebration/graduation planned
- [ ] Certification/credential if applicable
- [ ] Alumni community/next steps
- [ ] Testimonial/case study collection

## 4. Operations Readiness

### 4.1 Platform & Tools
- [ ] Course platform tested
- [ ] Video/live session tech tested
- [ ] Community platform ready
- [ ] Email sequences loaded

### 4.2 Team
- [ ] Instructor(s) confirmed
- [ ] TA/community manager if needed
- [ ] Support escalation path clear
- [ ] Backup plans for emergencies

### 4.3 Communication
- [ ] Pre-course emails scheduled
- [ ] Weekly reminder cadence set
- [ ] Announcement templates ready
- [ ] FAQ document prepared

## 5. Measurement Setup

### 5.1 Completion Tracking
- [ ] Completion defined clearly
- [ ] Progress tracking implemented
- [ ] Funnel stages identified
- [ ] Baseline documented

### 5.2 Learning Outcomes
- [ ] Pre/post assessment designed
- [ ] NPS/satisfaction survey ready
- [ ] Outcome achievement survey ready
- [ ] Testimonial collection process

### 5.3 Iteration Data
- [ ] Session engagement tracked
- [ ] Assignment completion tracked
- [ ] Drop-off points identified
- [ ] Feedback mechanisms in place
