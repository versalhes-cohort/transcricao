## Story

**As** an end user,
**I want** to paste a video URL and have its format and length pre-validated before any download begins,
**so that** I get fast feedback on bad inputs and never wait through a long download just to discover the video is too long.

## Acceptance Criteria

> Source: `docs/prd.md` Epic 1, Story 1.2 (verbatim).

1. **AC1:** The Streamlit page presents a single text input labeled "Cole aqui o link do v√≠deo" and a submit button labeled "Transcrever".
2. **AC2:** On submit, the URL is validated against an HTTP/HTTPS schema-only allowlist; malformed URLs trigger an immediate inline PT-BR error ("URL inv√°lida. Cole um link come√ßando com http:// ou https://.") **without backend processing**.
3. **AC3:** For valid URLs, the system invokes `yt-dlp` in metadata-only mode (`extract_info(download=False)`) to retrieve the video duration without downloading audio.
4. **AC4:** If `yt-dlp` cannot resolve the URL (unsupported source, unreachable, blocked), a PT-BR error is displayed: "N√£o foi poss√≠vel acessar este v√≠deo. Verifique o link ou tente outra fonte."
5. **AC5:** If the resolved duration exceeds 60 minutes, the system displays "V√≠deo excede o limite de 60 minutos. Tente um v√≠deo mais curto." and does not proceed.
6. **AC6:** If the resolved duration is ‚â§60 min, the UI transitions to a status state showing "Preparando download...", and Story 1.3's pipeline takes over.
7. **AC7:** All `yt-dlp` invocations use the Python API (`yt_dlp.YoutubeDL`); **no subprocess shell calls**.
8. **AC8:** Unit tests cover URL validation logic (valid/invalid schemas, empty input, whitespace).

### Task 2: Implement URL validation as pure function (AC: 2, 7, 8)

- [x] Create `app/url_validation.py` with a single public function:
  ```python
  from urllib.parse import urlparse
  from app.errors import InvalidURLError

  def validate_url(raw_url: str) -> str:
      """Validate a URL against the http/https schema-only allowlist.

      Returns the trimmed URL on success.
      Raises InvalidURLError with PT-BR message on failure.
      """
      if not raw_url or not raw_url.strip():
          raise InvalidURLError()
      url = raw_url.strip()
      try:
          parsed = urlparse(url)
      except Exception:
          raise InvalidURLError()
      if parsed.scheme not in ("http", "https"):
          raise InvalidURLError()
      if not parsed.netloc:
          raise InvalidURLError()
      return url
  ```

### Task 4: Implement `app/extractor.py` ‚Äî `fetch_metadata` only (AC: 3, 4, 5, 7)

> Architecture ¬ß4.3 describes `extractor.py` with both `fetch_metadata` AND `download_audio`. Story 1.2 only owns `fetch_metadata`. Story 1.3 will add `download_audio`.

- [x] Create `app/extractor.py` with:
  ```python
  from dataclasses import dataclass
  from urllib.parse import urlparse
  import yt_dlp
  from yt_dlp.utils import DownloadError as YtdlpDownloadError, ExtractorError
  from app.errors import UnsupportedSourceError, VideoTooLongError

  MAX_DURATION_SECONDS = 60 * 60  # 60 minutes ‚Äî FR4 hard cap

  @dataclass(frozen=True)
  class VideoMetadata:
      duration_seconds: int
      source_domain: str

  def fetch_metadata(url: str) -> VideoMetadata:
      """Fetch video metadata via yt-dlp without downloading audio.

      Raises:
          UnsupportedSourceError: yt-dlp cannot resolve the URL.
          VideoTooLongError: video duration > 60 minutes.
      """
      ydl_opts = {"quiet": True, "no_warnings": True, "skip_download": True}
      try:
          with yt_dlp.YoutubeDL(ydl_opts) as ydl:
              info = ydl.extract_info(url, download=False)
      except (YtdlpDownloadError, ExtractorError):
          raise UnsupportedSourceError()
      except Exception:
          # Be conservative: any other yt-dlp failure ‚Üí unsupported source
          raise UnsupportedSourceError()

      duration = info.get("duration") if info else None
      if duration is None or not isinstance(duration, (int, float)):
          raise UnsupportedSourceError()
      duration_int = int(duration)
      if duration_int > MAX_DURATION_SECONDS:
          raise VideoTooLongError()

      source_domain = urlparse(url).netloc or "unknown"
      return VideoMetadata(duration_seconds=duration_int, source_domain=source_domain)
  ```

### Architecture Decision (Story-Level)

`app/url_validation.py` is a **third utility module** added beyond the architecture's enumerated list in ¬ß4.1. The architecture v1.1 explicitly justified `chunking.py` and `logging_config.py` as additions; `url_validation.py` follows the same pattern (pure, single-purpose, easily testable).

**Rationale:**
- AC8 mandates testable URL validation logic.
- Putting validation in `app/main.py` is awkward to test ‚Äî Streamlit imports run module-level side effects.
- Putting validation inside `app/extractor.py` would mix URL string validation with yt-dlp interaction (separation of concerns violation).
- A new pure module (~25 lines) is the cleanest path.

**This is a non-breaking architectural extension** (no deletions, no behavior changes to existing modules). Flagging here so @po validates and @qa knows what to expect.
