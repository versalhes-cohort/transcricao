# Story 1.1: Project Foundation & Railway Deployment Canary

**InProgress**

## Story

**As** the project owner,
**I want** a minimal Streamlit application deployed and publicly accessible on Railway,
**so that** subsequent stories can incrementally add features against a working deployment baseline.

## Acceptance Criteria

> Source: `docs/prd.md` Epic 1, Story 1.1 (verbatim).

1. **AC1:** Repository initialized with `app/main.py` containing a minimal Streamlit page displaying "Transcrição Universal — pronto para uso" or equivalent placeholder text.
2. **AC2:** `requirements.txt` includes pinned versions of `streamlit`, `openai`, `yt-dlp`, `pydub` (or chosen chunking library), and any other direct dependencies.
3. **AC3:** `Dockerfile` (or `nixpacks.toml`) builds a Python 3.11+ container with `ffmpeg` installed as a system package and Streamlit configured to listen on Railway's expected port.
4. **AC4:** `.env.example` lists required environment variables (`OPENAI_API_KEY` minimum) without real values; actual `.env` is gitignored.
5. **AC5:** `railway.toml` (or Railway dashboard configuration) provisions a single service from the repository with HTTPS enabled.
6. **AC6:** The deployed URL renders the placeholder page over HTTPS within 60 seconds of cold start.
7. **AC7:** Railway healthcheck passes against the Streamlit process (HTTP 200 on the Streamlit endpoint).
8. **AC8:** A `README.md` documents local development (`streamlit run app/main.py`), deployment (Railway CLI or git-push), and required environment variables.
9. **AC9:** OpenAI account spending cap of USD $20/month is verified configured (manual stakeholder check; documented in README).

### Task 3: Create `Dockerfile` with Python 3.11 + ffmpeg (AC: 3)

- [x] At repo root, create `Dockerfile` matching the architecture skeleton **exactly**:
  ```dockerfile
  FROM python:3.11-slim

  RUN apt-get update && apt-get install -y --no-install-recommends \
      ffmpeg \
      && rm -rf /var/lib/apt/lists/*

  WORKDIR /app

  COPY requirements.txt .
  RUN pip install --no-cache-dir -r requirements.txt

  COPY app/ ./app/

  ENV STREAMLIT_SERVER_HEADLESS=true \
      STREAMLIT_SERVER_ADDRESS=0.0.0.0 \
      STREAMLIT_BROWSER_GATHER_USAGE_STATS=false

  EXPOSE 8501
  CMD streamlit run app/main.py --server.port=${PORT:-8501}
  ```

### Stack Constraints

- **Python:** 3.11+ required (architecture pins to `python:3.11-slim` Docker base). Local dev system has Python 3.12.3 — compatible.
  - [Source: `docs/architecture.md` §2.3]
- **Streamlit:** 1.40.0 pinned. Version selected for stable WebSocket session model that holds during multi-minute processing (NFR14).
  - [Source: `docs/architecture.md` §2.4 + §7]
- **System dep:** `ffmpeg` MUST be installed in the Docker image. Used by yt-dlp post-processing (Story 1.3) and pydub chunking (Story 1.4). Pre-install in Story 1.1 even though not yet exercised — Story 1.1 is the deployment canary that proves the image build works end-to-end.
  - [Source: `docs/architecture.md` §2.3]

### Module Boundary (Story 1.1 scope)

Per the canonical layout, Story 1.1 creates **only**:
- `app/__init__.py` (empty)
- `app/main.py` (minimal Streamlit page — no business logic)

Do **NOT** create the following modules in this story (each owned by a later story):
- `app/extractor.py` → Story 1.3
- `app/transcriber.py` → Story 1.4
- `app/chunking.py` → Story 1.4
- `app/duration.py` → Story 1.5
- `app/errors.py` → Story 1.6
- `app/logging_config.py` → Story 1.3 (defense-in-depth for NFR10)

  - [Source: `docs/architecture.md` §4.1 + §4.2]

### Deployment Topology Anchors

- **Single Railway service**, single Docker container, no DB, no queue, no auth.
- HTTPS provided by Railway default ingress (no app code needed).
- Healthcheck path: `/_stcore/health` (Streamlit native; **never** write a custom route).
- Cold-start budget: ≤60s (NFR16); ~30s headroom expected per architecture §2.7.
- `$PORT` is auto-injected by Railway; Dockerfile CMD uses `${PORT:-8501}` so 8501 is the local default.
  - [Source: `docs/architecture.md` §2.1, §2.3, §2.6, §2.7; PRD NFR4–5, NFR16]

### Security/Secrets Anchors (CRITICAL — repo is public)

- The repo `versalhes-cohort/transcricao` is **public**. Risk surface:
  - `.env` MUST stay gitignored. Already verified in bootstrap. Re-verify in Task 4.
  - `OPENAI_API_KEY` ONLY lives in Railway dashboard env vars, never in code, never in committed files.
  - `.env.example` is committed but contains empty `OPENAI_API_KEY=` (no value).
  - The boot-time check for `OPENAI_API_KEY` is **not** required in Story 1.1 (Story 1.4 owns it). Story 1.1's `app/main.py` does not call OpenAI yet.
  - [Source: `docs/architecture.md` §2.5, §8.3; PRD NFR6]

### Cost-Cap Anchor (NFR3)

- The OpenAI account spending cap (USD $20/month) is enforced **outside the application** at https://platform.openai.com/settings/organization/limits. Application code does NOT track spending.
- AC9 is a **stakeholder action**, not a code task. The dev agent cannot complete this AC autonomously — confirm with user.
