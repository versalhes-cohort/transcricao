## Story

**As** an end user,
**I want** the audio to be transcribed in PT-BR without me worrying about file size limits,
**so that** I get a complete transcription regardless of whether the video is 5 minutes or 60 minutes long.

## Acceptance Criteria

> Source: `docs/prd.md` Epic 1, Story 1.4 (verbatim).

1. **AC1:** The system reads the audio file produced by Story 1.3 and inspects its size in bytes.
2. **AC2:** If the file is **≤25 MB**, it is sent to OpenAI's `whisper-1` API in a single request with `language="pt"` and `response_format="text"` (or equivalent plain-text response).
3. **AC3:** If the file is **>25 MB**, the system splits the audio into sequential chunks of **≤24 MB each (1 MB safety margin)** using `pydub` or `ffmpeg`, with a **2-second overlap** at chunk boundaries to prevent words being cut mid-utterance.
4. **AC4:** Each chunk is transcribed independently via the Whisper API with `language="pt"`. Chunk transcripts are then concatenated in chronological order to produce the final transcript.
5. **AC5:** When chunks overlap, the overlap region is de-duplicated using a simple sentence/word-boundary heuristic; **minor duplication at chunk seams is acceptable for MVP** if no clean boundary is found.
6. **AC6:** If the Whisper API returns an error (auth failure, rate limit, server error), the system retries up to **2 times with exponential backoff** before surfacing a PT-BR error: **"Falha na transcrição. Tente novamente em alguns minutos."**
7. **AC7:** The final transcript text is held in memory only (no disk persistence) and passed to Story 1.5's rendering step.
8. **AC8:** Unit tests cover chunk-boundary math (correct chunk count for various file sizes, correct overlap, correct stitching of mock transcripts).
9. **AC9:** The OpenAI API key is read from `os.environ["OPENAI_API_KEY"]`; **missing key raises a clear configuration error at app startup, not at first request.**

### Task 2: Create `app/chunking.py` — pure boundary math + stitching (AC: 3, 4, 5, 8)

> The architecture's most-justified utility module split: pure boundary math separated from audio I/O so it can be unit-tested in isolation. Architecture §4.2 + §10 ADR-006 explicitly justify this.

- [x] Create `app/chunking.py` with two pure functions and a `ChunkSpec` dataclass:
  ```python
  from dataclasses import dataclass

  @dataclass(frozen=True)
  class ChunkSpec:
      """Time range for one audio chunk, in milliseconds."""
      start_ms: int
      end_ms: int


  def plan_chunks(
      file_size_bytes: int,
      audio_duration_ms: int,
      max_chunk_bytes: int = 24 * 1024 * 1024,
      overlap_ms: int = 2000,
  ) -> list[ChunkSpec]:
      """Compute chunk boundaries (in ms) that respect the Whisper 25 MB
      single-request limit.

      Algorithm (architecture §5.2):
      1. Approximate ms-per-byte assuming uniform CBR encoding.
      2. Compute target chunk duration in ms.
      3. Effective stride = chunk_duration - overlap.
      4. Slide forward, clamping the final chunk to audio_duration_ms.
      """
      ms_per_byte = audio_duration_ms / file_size_bytes
      chunk_duration_ms = int(max_chunk_bytes * ms_per_byte)
      stride_ms = chunk_duration_ms - overlap_ms

      chunks: list[ChunkSpec] = []
      cursor_ms = 0
      while cursor_ms < audio_duration_ms:
          end_ms = min(cursor_ms + chunk_duration_ms, audio_duration_ms)
          chunks.append(ChunkSpec(start_ms=cursor_ms, end_ms=end_ms))
          if end_ms >= audio_duration_ms:
              break
          cursor_ms += stride_ms
      return chunks


  def stitch_transcripts(chunks: list[str], max_overlap_words: int = 25) -> str:
      """Concatenate chunk transcripts with simple boundary de-duplication.

      Strategy (architecture §5.3):
      - For each adjacent pair, find the longest suffix-of-tail / prefix-of-head
        n-gram (case-insensitive, normalized punctuation).
      - If overlap >= 3 words → elide; else concat with single space.
      """
      if not chunks:
          return ""
      if len(chunks) == 1:
          return chunks[0]
      result = chunks[0]
      for next_chunk in chunks[1:]:
          result = _merge_at_boundary(result, next_chunk, max_overlap_words)
      return result


  def _merge_at_boundary(left: str, right: str, max_overlap_words: int) -> str:
      """Find the longest matching word-sequence at the seam and elide.

      Returns the merged string. If no overlap of >=3 words is found,
      returns left + ' ' + right (per AC5: minor duplication acceptable).
      """
      tail_words = left.split()[-max_overlap_words:]
      head_words = right.split()[:max_overlap_words]

      # Normalize for matching (case-insensitive, strip trailing punctuation).
      def norm(w: str) -> str:
          return w.strip(".,;:!?\"'").lower()

      tail_norm = [norm(w) for w in tail_words]
      head_norm = [norm(w) for w in head_words]

      # Find longest k where tail_norm[-k:] == head_norm[:k].
      best_k = 0
      for k in range(min(len(tail_norm), len(head_norm)), 0, -1):
          if tail_norm[-k:] == head_norm[:k]:
              best_k = k
              break

      if best_k >= 3:
          # Elide overlap: keep `left`, append `right` from after the overlap.
          remainder = " ".join(right.split()[best_k:])
          return f"{left} {remainder}".strip() if remainder else left

      # Fallback: simple concat with space (AC5 allows minor duplication).
      return f"{left} {right}"
  ```
  - [Source: `docs/architecture.md` §5.2 + §5.3]

### Task 3: Create `app/transcriber.py` — Whisper orchestration (AC: 1, 2, 3, 4, 6, 7, 9)

- [x] Create `app/transcriber.py` with these public functions and module logger:
  ```python
  import logging
  import os
  import time
  from pathlib import Path

  import openai
  from pydub import AudioSegment

  from app.chunking import plan_chunks, stitch_transcripts
  from app.errors import TranscriptionError

  logger = logging.getLogger(__name__)

  WHISPER_SINGLE_CALL_LIMIT_BYTES = 25 * 1024 * 1024  # OpenAI's 25 MB hard limit
  CHUNK_TARGET_BYTES = 24 * 1024 * 1024              # 1 MB safety margin
  WHISPER_LANGUAGE = "pt"
  WHISPER_MODEL = "whisper-1"
  RETRY_MAX_ATTEMPTS = 3                              # initial + 2 retries (AC6)
  RETRY_BASE_DELAY_S = 1.0                            # exponential: 1, 2, 4...
  EMPTY_TRANSCRIPT_THRESHOLD = 10                     # chars (architecture §12 R8)


  def assert_api_key_configured() -> None:
      """Boot-time check (AC9). Raises RuntimeError if OPENAI_API_KEY is missing.

      Called from main.py at module load — BEFORE any user request can hit
      the pipeline. Failure here causes Streamlit startup to fail loudly,
      which is the desired behavior (fail fast, fail visible).
      """
      if not os.environ.get("OPENAI_API_KEY"):
          raise RuntimeError(
              "OPENAI_API_KEY environment variable is required. "
              "Set it in Railway dashboard → Variables, or in a local .env file."
          )


  def transcribe(audio_path: Path, audio_duration_s: int) -> str:
      """Transcribe an audio file in PT-BR via OpenAI Whisper.

      Dispatches to single-call or chunked path based on file size.

      Args:
          audio_path: Path to the mp3/m4a audio file (from Story 1.3).
          audio_duration_s: Total duration in seconds (from fetch_metadata).

      Returns:
          Final transcript as plain string, in chronological order.

      Raises:
          TranscriptionError: Whisper API failed after all retries OR
              the final transcript is suspiciously empty (<10 chars).
      """
      file_size_bytes = audio_path.stat().st_size

      if file_size_bytes <= WHISPER_SINGLE_CALL_LIMIT_BYTES:
          transcript = _transcribe_single(audio_path)
      else:
          transcript = _transcribe_chunked(audio_path, audio_duration_s)

      # Architecture §12 R8: if final transcript is suspiciously empty,
      # surface as TranscriptionError rather than render a blank result.
      if len(transcript.strip()) < EMPTY_TRANSCRIPT_THRESHOLD:
          raise TranscriptionError()

      return transcript


  def _transcribe_single(audio_path: Path) -> str:
      """Single-call path (file ≤25 MB). Uses 2-retry exponential backoff."""
      def _call():
          with audio_path.open("rb") as fh:
              return openai.audio.transcriptions.create(
                  model=WHISPER_MODEL,
                  file=fh,
                  language=WHISPER_LANGUAGE,
                  response_format="text",
              )
      return _with_retries(_call)


  def _transcribe_chunked(audio_path: Path, audio_duration_s: int) -> str:
      """Chunked path (file >25 MB). Architecture §5.2 algorithm."""
      file_size_bytes = audio_path.stat().st_size
      audio_duration_ms = audio_duration_s * 1000
      chunk_specs = plan_chunks(
          file_size_bytes=file_size_bytes,
          audio_duration_ms=audio_duration_ms,
          max_chunk_bytes=CHUNK_TARGET_BYTES,
          overlap_ms=2000,
      )

      chunk_dir = audio_path.parent  # caller owns the dir; clean handled by main.py
      audio = AudioSegment.from_file(audio_path)
      transcripts: list[str] = []
      for idx, spec in enumerate(chunk_specs):
          chunk_path = chunk_dir / f"chunk_{idx:03d}.mp3"
          audio[spec.start_ms : spec.end_ms].export(
              chunk_path, format="mp3", bitrate="64k"
          )

          def _call():
              with chunk_path.open("rb") as fh:
                  return openai.audio.transcriptions.create(
                      model=WHISPER_MODEL,
                      file=fh,
                      language=WHISPER_LANGUAGE,
                      response_format="text",
                  )
          transcripts.append(_with_retries(_call))

      return stitch_transcripts(transcripts)


  def _with_retries(call):
      """Execute `call` with up to 2 retries on retryable errors (AC6)."""
      last_exc: Exception | None = None
      for attempt in range(RETRY_MAX_ATTEMPTS):
          try:
              return call()
          except openai.AuthenticationError:
              # Auth errors are NOT retryable — fail fast.
              raise TranscriptionError()
          except openai.APIError as e:
              last_exc = e
              if attempt < RETRY_MAX_ATTEMPTS - 1:
                  time.sleep(RETRY_BASE_DELAY_S * (2 ** attempt))
                  continue
              raise TranscriptionError()
          except Exception as e:
              # Be conservative: any unexpected error after retries → TranscriptionError
              last_exc = e
              if attempt < RETRY_MAX_ATTEMPTS - 1:
                  time.sleep(RETRY_BASE_DELAY_S * (2 ** attempt))
                  continue
              raise TranscriptionError()
      raise TranscriptionError()  # unreachable, but satisfies type-checker
  ```
  - [Source: `docs/architecture.md` §3.1 stage 6, §4.3, §5.2, §6.2, §12 R8]

### Audio Chunking Algorithm (architecture §5.2 — verbatim)

The implementation in Task 2 is byte-identical to the architecture's worked algorithm in §5.2:

| Parameter | Default | Source |
|---|---|---|
| `max_chunk_bytes` | `24 * 1024 * 1024` (24 MB) | architecture §5.2 — 1 MB safety margin under Whisper's 25 MB |
| `overlap_ms` | `2000` (2 s) | architecture §5.2 — Story 1.4 AC3 |
| `ms_per_byte` | `audio_duration_ms / file_size_bytes` | architecture §5.2 — assumes uniform CBR |

Worked example (60-min video, 28 MB file):
- ms_per_byte = 3,600,000 / 29,360,128 ≈ 0.1226
- chunk_duration_ms = 24×1024×1024 × 0.1226 ≈ 3,084,000 ms (~51.4 min)
- 2 chunks: [0–3,084,000], [3,082,000–3,600,000]

  - [Source: `docs/architecture.md` §5.2]

### Stitching Algorithm (architecture §5.3 — verbatim)

The merge-at-boundary heuristic in Task 2:
1. Take last 25 words of `tail`, first 25 words of `head`.
2. Find longest suffix-of-tail that is prefix-of-head (case-insensitive, normalized punctuation).
3. If overlap ≥3 words → elide; else concat with space (AC5 fallback).

> "minor duplication at chunk seams is acceptable for MVP if no clean boundary is found." — PRD AC5

  - [Source: `docs/architecture.md` §5.3 + PRD AC5]

### Retry Strategy (AC6)

- Total attempts: 3 (initial + 2 retries)
- Backoff: exponential — 1s, 2s, 4s (only the last 2 sleeps actually fire)
- Retryable: `openai.APIError` (rate limit, 5xx)
- NOT retryable: `openai.AuthenticationError` (fail fast — bad key won't fix itself)
- Outcome on exhaustion: `TranscriptionError` (verbatim PT-BR message)

  - [Source: PRD AC6; architecture §6.2]

### Empty-Transcript Safety (architecture §12 R8)

The architect resolved R8 in Story 1.1 closure:

Implementation: at the end of `transcribe()`, `if len(transcript.strip()) < 10: raise TranscriptionError()`. This handles silent-audio + chunk-all-empty edge cases without rendering a blank UI to the user.

#### INFO-only Observations (advisory, for @dev awareness)

These are stylistic/defensive improvements that @dev MAY apply but don't block ship. Document in Dev Notes if applied.

**INFO-1 (LOW): Defensive guard in `plan_chunks` for hypothetical `stride_ms <= 0`**

If `max_chunk_bytes * ms_per_byte <= overlap_ms`, the algorithm would have `stride_ms <= 0` → infinite loop. With the production defaults (24 MB chunks, 2-s overlap, ≤60-min audio), this is **mathematically impossible** (would require ~100 Mbps audio). But a defensive `if stride_ms <= 0: stride_ms = chunk_duration_ms` (no overlap) would be cheap insurance against future param changes.

**Recommendation:** add the guard. Cost: 2 lines. Benefit: hardens the function for future maintainers.

**INFO-2 (LOW): Dead-code `last_exc` variable in `_with_retries`**

The variable is assigned but never used. Either:
- Remove it (cleanest), OR
- Use it: `raise TranscriptionError() from last_exc` to preserve the cause chain

**Recommendation:** option B (preserve cause chain). Helps future debugging when smoke tests fail.

**INFO-3 (LOW): Retry duplicate code paths**

`_with_retries` has two nearly-identical except clauses (`openai.APIError` and `Exception`). Could consolidate:

```python
except openai.AuthenticationError:
    raise TranscriptionError()  # never retry auth errors
except (openai.APIError, Exception) as e:
    last_exc = e
    if attempt < RETRY_MAX_ATTEMPTS - 1:
        time.sleep(RETRY_BASE_DELAY_S * (2 ** attempt))
        continue
    raise TranscriptionError() from last_exc
```

**Recommendation:** apply if straightforward; otherwise leave for future cleanup.

**INFO-4 (LOW): `25 * 1024 * 1024` (binary MiB) interpretation**

OpenAI's "25 MB" limit is ambiguous (decimal vs binary). Story uses binary (26,214,400 bytes). If Whisper actually rejects files between 25,000,000 and 26,214,400 bytes, files in that range would fail with API error before chunking kicks in.

**Mitigation in place:** the chunk-target is `24 * 1024 * 1024` (24 MiB), so once we DO chunk, we're safe. The risk is ONLY for files in the narrow 25,000,000–26,214,400 byte range that we'd send single-call.

**Recommendation:** leave as-is for now. Smoke testing in production will catch any real issue. If smoke fails on edge-sized files, change `WHISPER_SINGLE_CALL_LIMIT_BYTES = 25 * 1000 * 1000` (decimal MB) — narrows the gap to chunk-target.

**INFO-5 (LOW): `pytest.raises(Exception)` is too broad**

In `test_chunk_specs_are_frozen`, the assertion catches ANY exception. More precise: `pytest.raises(dataclasses.FrozenInstanceError)`. Trivial.
