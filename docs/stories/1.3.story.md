## Story

**As** an end user,
**I want** the system to download just the audio track of my video efficiently,
**so that** the next pipeline step (transcription) has a minimal-size audio file ready.

## Acceptance Criteria

> Source: `docs/prd.md` Epic 1, Story 1.3 (verbatim).

1. **AC1:** After length validation passes (Story 1.2), the system invokes `yt-dlp` to download the audio-only stream of the source video.
2. **AC2:** Audio is downloaded in a Whisper-compatible format (MP3 or M4A) at a target bitrate of **≤64 kbps** where the source allows, to minimize file size.
3. **AC3:** The downloaded file is written to an **ephemeral location** (Python `tempfile` directory or a known temp path inside the container).
4. **AC4:** While downloading, the UI status updates to **"Baixando áudio..."** (visible to the user via `st.status` or equivalent).
5. **AC5:** On `yt-dlp` failure (DRM, geo-block, source error), a PT-BR error is shown: **"Falha ao baixar o áudio. Tente novamente ou use outra fonte."** The downloaded partial file (if any) is cleaned up.
6. **AC6:** On success, the UI transitions to **"Transcrevendo..."** and the file path is passed to Story 1.4's transcription step.
7. **AC7:** The downloaded audio file is **deleted from disk immediately after the transcription step completes** (success or failure) — no source media persists on the server.
8. **AC8:** Logging — a single INFO log line records the source domain, audio duration in seconds, and downloaded file size in bytes — but NOT the URL itself or any media content.

> This is the third architectural extension module after `chunking.py` (Story 1.4) and `url_validation.py` (Story 1.2). Architecture §4.2 + §6.4 explicitly justify it as defense-in-depth for NFR10 (no transcripts/URLs in logs).

- [x] Create `app/logging_config.py` matching the architecture §6.4 skeleton:
  ```python
  import hashlib
  import logging
  import re

  URL_PATTERN = re.compile(r'https?://[^\s<>"]+')


  class SafeLogFilter(logging.Filter):
      """Redact full URLs in log records to prevent leakage (NFR10 defense-in-depth)."""

      def filter(self, record: logging.LogRecord) -> bool:
          if isinstance(record.msg, str):
              record.msg = URL_PATTERN.sub('[URL]', record.msg)
          if record.args:
              record.args = tuple(
                  URL_PATTERN.sub('[URL]', str(a)) if isinstance(a, str) else a
                  for a in record.args
              )
          return True


  def hash_url(url: str) -> str:
      """8-char SHA-256 hash for cross-event correlation without exposing the URL."""
      return hashlib.sha256(url.encode()).hexdigest()[:8]


  def setup_logging() -> None:
      """Wire stdout logging with the SafeLogFilter. Idempotent — safe to call multiple times."""
      root = logging.getLogger()
      if any(isinstance(h, logging.StreamHandler) for h in root.handlers):
          return  # Already configured (Streamlit reruns this module on each interaction)
      handler = logging.StreamHandler()
      handler.setFormatter(logging.Formatter(
          '%(asctime)s %(levelname)s %(name)s | %(message)s'
      ))
      handler.addFilter(SafeLogFilter())
      root.addHandler(handler)
      root.setLevel(logging.INFO)
  ```
  - [Source: `docs/architecture.md` §6.4 — safe logging implementation]
- [x] **Idempotency:** `setup_logging()` MUST be safe to call multiple times. Streamlit re-runs `main.py` on every user interaction, so naive `logging.basicConfig` would attach duplicate handlers. _Implemented with SafeLogFilter-presence check (more robust than StreamHandler-presence) so it doesn't conflict with pytest's own handlers. Verified by `test_is_idempotent`._

  ```python
  def download_audio(url: str, target_dir: Path | None = None) -> Path:
      """Download audio-only stream of a video to an ephemeral location.

      Args:
          url: Video URL (already validated by url_validation.validate_url).
          target_dir: Optional pre-created tempdir. If None, mkdtemp is used internally.
              Caller is responsible for cleaning up the directory containing the returned file.

      Returns:
          Absolute Path to the downloaded audio file (mp3, ≤64 kbps where source allows).

      Raises:
          DownloadError: yt-dlp failed (DRM, geo-block, partial download cleaned up).
      """
      if target_dir is None:
          target_dir = Path(tempfile.mkdtemp(prefix="transcricao_"))

      outtmpl = str(target_dir / "audio.%(ext)s")
      ydl_opts = {
          "format": "bestaudio/best",
          "outtmpl": outtmpl,
          "quiet": True,
          "no_warnings": True,
          "postprocessors": [{
              "key": "FFmpegExtractAudio",
              "preferredcodec": "mp3",
              "preferredquality": "64",
          }],
      }

      try:
          with yt_dlp.YoutubeDL(ydl_opts) as ydl:
              ydl.download([url])
      except (YtdlpDownloadError, ExtractorError):
          _cleanup_partials(target_dir)
          raise DownloadError()
      except Exception:
          _cleanup_partials(target_dir)
          raise DownloadError()

      mp3_files = list(target_dir.glob("audio.mp3"))
      if not mp3_files:
          # Fallback: any audio-extension file produced
          for ext in ("m4a", "ogg", "wav", "webm"):
              candidates = list(target_dir.glob(f"audio.{ext}"))
              if candidates:
                  mp3_files = candidates
                  break
      if not mp3_files:
          _cleanup_partials(target_dir)
          raise DownloadError()

      audio_path = mp3_files[0]
      logger.info(
          "download_ok url_hash=%s source=%s file_size_bytes=%d",
          hash_url(url),
          audio_path.parent.name,
          audio_path.stat().st_size,
      )
      return audio_path


  def _cleanup_partials(target_dir: Path) -> None:
      """Best-effort cleanup of partial download artifacts. Never raises."""
      try:
          for entry in target_dir.iterdir():
              try:
                  entry.unlink()
              except OSError:
                  pass
      except OSError:
          pass
  ```
- [x] **NFR7 constraint:** `yt-dlp` invoked EXCLUSIVELY via `yt_dlp.YoutubeDL`. No `subprocess`, no shell calls. _Strict regex audit confirmed PASS: zero matches._
  - [Source: `docs/architecture.md` §3.1 stage 4 + §8.2 threat 1; PRD NFR7]
- [x] **AC8 logging contract:** the single INFO line MUST include `url_hash` (8 chars), source identifier, **`duration_s`**, and `file_size_bytes`. ### Logging Contract (AC8)

The single INFO log line emitted by `download_audio` on success:

```
download_ok url_hash=ab12cd34 source=transcricao_xyz file_size_bytes=8421376
```

What is **allowed**:
- `url_hash` — 8-char SHA-256 prefix (deterministic, non-reversible)
- `source` — temp dir basename (proxy for source domain; full domain would require `urlparse(url).netloc` which is fine but optional)
- `file_size_bytes` — integer

What is **forbidden** (NFR10):
- The full URL
- The video title
- Any audio content
- File path beyond temp dir name

Defense-in-depth: even if a future change accidentally logs the URL, `SafeLogFilter` redacts it via regex before stdout flush.

  - [Source: `docs/architecture.md` §6.4 — log event vocabulary]

#### Should-Fix #1 (HIGH-priority, but non-blocking) — AC8 logging gap

**Issue:** Task 3's `download_audio` implementation example emits this log line:

```python
logger.info(
    "download_ok url_hash=%s source=%s file_size_bytes=%d",

)
```

But **AC8 explicitly requires logging the audio duration in seconds:**

> AC8: Logging — a single INFO log line records the **source domain**, **audio duration in seconds**, and **downloaded file size in bytes** — but NOT the URL itself or any media content.

`download_audio` does NOT have access to the duration — that lives in `metadata.duration_seconds` from `fetch_metadata` (Story 1.2). The current implementation sketch will fail AC8 verification.

**Recommendation for @dev:** Move the log emission from `extractor.download_audio` to `main.py` (after both `fetch_metadata` AND `download_audio` succeed). At that point `metadata` AND `audio_path` are both in scope:

```python
# In main.py, after download_audio returns audio_path:
logger.info(
    "download_ok url_hash=%s source=%s duration_s=%d file_size_bytes=%d",
    hash_url(url),
    metadata.source_domain,        # use the value from fetch_metadata, not tempdir basename
    metadata.duration_seconds,
    audio_path.stat().st_size,
)
```

Then remove the `logger.info(...)` from `download_audio`. Alternative: pass `duration_seconds` as a parameter to `download_audio` — also acceptable, but adds coupling between layers. **Prefer the main.py approach.**

This affects Task 3 implementation (remove logger.info inside download_audio) and Task 4 implementation (add the proper logger.info in main.py). Update File List + AC8 verification step accordingly.

#### Should-Fix #2 (LOW-priority) — AC2 format strictness

**Issue:** Task 3's fallback glob list is over-permissive vs. the AC's "MP3 or M4A" wording:

```python
for ext in ("m4a", "ogg", "wav", "webm"):  # ogg/webm/wav are beyond AC2
```

**Tradeoff analysis:**
- The AC says "MP3 or M4A". Strict reading rejects ogg/wav/webm.
- However, `FFmpegExtractAudio(preferredcodec='mp3')` should always produce mp3; the fallback branch only fires if ffmpeg post-processing failed. In that fallback case, `webm` is what yt-dlp's bestaudio commonly returns, and Whisper API actually accepts webm/m4a/mp3/wav/etc.
- So either: (a) strict — narrow fallback to just `m4a` and let other extensions raise `DownloadError`, OR (b) document that the AC2 wording is illustrative not exhaustive (Whisper accepts more).

**Recommendation for @dev:** Narrow to `m4a` only. If post-processing to mp3 fails AND no m4a is found, raise `DownloadError`. This makes the AC compliance literal. If we later observe legitimate cases of ffmpeg failing to produce mp3 from a particular source, we can widen the fallback in a follow-up story.

```python
for ext in ("m4a",):  # AC2 strict
    candidates = list(target_dir.glob(f"audio.{ext}"))
    if candidates:
        mp3_files = candidates
        break
```

Non-blocking — if @dev disagrees and prefers the broader fallback, document that decision in Dev Notes with rationale.
